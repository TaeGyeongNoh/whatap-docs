Second quarter guidance

July 9, 2024
TW J
Technical writer

Here’s an overview of new features and notable changes of WhaTap’s 2nd quarter services in 2024. Take a look at new features offered by WhaTap at a glance.

Common

Common menu improvements have been made for better convenience.

Application
  • Transaction Map has been further improved following the previous quarter.
  • A default preset changing feature has been added to Application dashboard.
Server
  • The installation screen has been reorganized to allow you to select the OS first when creating a server project.
  • The Unix Oracle Solaris installation package has been provided.
Kubernetes
  • The Agent Installation screen has been reorganized to provide an easier method to select options.
  • The Pending Pod Status menu has been newly reorganized.
  • In the Container Map menu, Resource Relationship Map has been provided.
Database

The Oracle Pro Beta service is available.

Browser

A new Session replay feature has been provided.

An overview of changes other than new features for each product, the corresponding release versions, and changes in the agent can be found in the following guide.

Application​

Notable new features in the Application section of WhaTap in Q2 2024 include the improvements of Transaction Map and the feature to change the default preset of Application dashboard, following the previous quarter (Q1).

As part of improvements of Transaction Map, a new custom filtering feature has been provided that allows you to view only the selected items in the response time distribution chart. In the Custom tab next to the TOP 30 section, enter the desired value directly for a specific transaction, domain, or IP to search for, or select an item from the TOP 30 list. The response time distribution of the selected items is filtered in real time on the chart.

You can change the default dashboard preset that can be viewed when you first enter the Application dashboard menu. It provides a more personalized user experience by setting the preferred dashboard configuration as the default preset. After modifying the widget selection and arrangement as desired, try using the Set as primary option in the preset selection box on the upper right of the dashboard. The dashboard designated as default is marked with the default tag in the preset list.

Server​

Notable new features in the Server section of WhaTap Q2 2024 include improvements to the installation screen and the introduction to the Unix Solaris installation package.

The installation process for server monitoring products has been improved to be more intuitive. Instead of the previous method of separating tabs for each OS, user can now select an OS first and then an individual installation page tailored to the OS appears.

In this Q2, a new Unix Solaris agent installation package is to be provided. Solaris that has been developed by Sun Microsystems, is a high-performance Unix operating system known for its scalability on SPARC systems and the features such as ZFS file system and DTrace. WhaTap supports Oracle Solaris 11.2 or later. You can install it simply by downloading the package by using the Solaris installation package and then running it. WhaTap Server Monitoring allows you to manage and monitor the Solaris operating system more effectively.

Kubernetes​

Notable new features and changes in the Kubernetes section of WhaTap Q2 2024 include the reorganization of Agent Installation and Pending Pod Status, and the introduction to the Resource relations feature in Container Map.

Due to ongoing changes in Kubernetes products and emergence of various container runtimes, the agent installation process is also becoming more complex. Following the support for Helm chart installation in Q1, WhaTap provide a new Kubernetes Agent Installation feature in Q2. Previously, after downloading the YAML file for installation, there were cases where the items needed to be modified additionally. But now, when you select key components or features in the Kubernetes environment and features to use, appropriate installation files are provided in the UI. This makes it much faster and easier to install agents.

The Pending Pod Status feature provided under the Laboratory menu has been reorganized into V2 and the sub menu of Workload has been officially released. The Pending Pod Status feature provides real-time visibility into a Pod in Pending status, along with the related data that enables root cause analysis. This helps increase the availability of clusters and services.

WhaTap Kubernetes Monitoring has undergone continuous Container Map improvements throughout the first half of 2024. In the second quarter, the Resource relations feature has been added to Container Map. The Resource relations feature allows you to visually check the hierarchy and relationships between Kubernetes resources, allowing you to analyze the Kubernetes environment and identify the cause and impact at a glance when a problem occurs.

Database​

WhaTap is now announcing notable new features and changes in the Database section in Q2 2024, including the launch of Oracle product V2 and the Beta launch of Oracle Pro.

The WhaTap database product suite is transitioning to V2. Following Redis and MongoDB in Q1, the transition to the Oracle V2 has begun in Q2.

The Oracle Pro Beta service has launched in preparation for the official release in Q3. Oracle Pro has applied the technologies to monitor the performance metrics by directly accessing the memory of the database server. It can monitor accurate performance information up to 20 times per second by reflecting the latest data, and can minimize the load on the system by directly accessing memory without using the disk I/O. This allows you to quickly and accurately analyze issues based on the precisely collected performance data, such as execution time for a specific query and status of the buffer cache.

Browser​

WhaTap introduces Session replay as a notable new feature in the Browser section for Q2 2024. It may be difficult to accurately reproduce the failures with just the list of events. The Session replay feature allows you to automatically record and play every event users perform on the website.

The interactions can be analyzed between the website and users by accurately reproducing every event, such as user clicks, scrolls, inputs, and page transitions. This enables real user experiences, allowing you to quickly identify and resolve potential issues, and provides insights into how users use the website and which can help you optimize it.

Feature In Page Load Hitmap, AJAX Hitmap, and User Session Log Search, added the Session replay feature. Service 2.6.0​
