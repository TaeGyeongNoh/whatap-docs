## Log management options

When the application server runs, it starts collecting the monitoring data. The collected data is stored in the log file.

- **log_rotation_enabled** <span class='type'>Boolean</span>

  Default `true`

  The function to store agent log files by date is activated. The log file name is stored in the format of _whatap-'yyyymmdd'.log_. If the value is `false`, the log file name is stored in the format of _whatap.log_.

- **log_keep_days** <span class='type'>Int</span>

  Default `7`

  It sets the log file retention period. It works only when `log_rotation_enabled` is `true`.

- **log_datasource_lookup_enabled** <span class='type'>Boolean</span>

  Default `true`

  The function to record the log is activated in case of DataSource upon InitialContext Lookup.

- **log_root** <span class='type'>String</span>

  You can set the path for the agent log files. The _logs_ folder is created under the specified path, and the WhaTap log files are generated in this folder.

  If you set an agent option as in the following example, the path for creating the whatap log file is as follows:

  > <em>/Users/sample_user/workspace/nodejs_application/whatap_logs/logs/</em>

  ```ini title="whatap.conf"
  log_root=/Users/sample_user/workspace/nodejs_application/whatap_logs/
  ```

## Setting the log sink

In [Node.js agent 0.5.1](../release-notes/nodejs/nodejs-0_5_1) or later, the LogSink feature is provided for integrated application server monitoring.

- **logsink_enabled** <span class='type'>Boolean</span>

  Default `false`

  Set whether or not to enable the log monitoring.

  :::note

  - For more information on how to enable the log monitoring, see [the following](../log/log-nodejs).

  - For more information about the log monitoring, see [the following](../log/learn-main-menu).

  :::

- **logsink_trace_txid_enabled** <span class='type'>Boolean</span>

  Default `true`

  The transaction ID is inserted into the log to configure the visibility of the <Cmdname sid="TTL06128" className="uitext" /> tab in <Cmdname sid="TTL06209" className="uitext" />.

  :::note

  For more information about the <Cmdname sid="TTL06209" className="uitext" />의 <Cmdname sid="TTL06128" className="uitext" /> tab, see [the following](dashboard-hitmap-trace#anlaysis-trace-transaction-log).

  :::

- **logsink_limit_content_enabled** <span class='type'>Boolean</span>

  Default `true`

  Set whether or not to limit the maximum size for log messages.

- **logsink_limit_content_length** <span class='type'>Number</span>

  Default `10,000`

  It sets the maximum length for log messages. This option applies only when the `logsink_limit_content_enabled` option is set to `true`.
