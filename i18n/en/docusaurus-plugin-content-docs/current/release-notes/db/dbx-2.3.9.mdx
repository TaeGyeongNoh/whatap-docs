February 18, 2025

## Tibero

- <Status priority='1'>Changed</Status> When sending the plan information, if `parent_id` is `null`, it is sent as `-1` (applied from version 20250220).

- <Status priority='0'>Fixed</Status> Tibero 5 (rev. 125305 to rev. 127709) Treated the status column name in `$session` as an error.

## Oracle

- <Status priority='1'>Feature</Status> Added the `!rectype` and `@id` fields in the following categories:

  > `db_oracle_active_session`, `db_oracle_xos_disk_usage`

- <Status priority='1'>Changed</Status> When sending the plan information, if `parent_id` is `null`, it is sent as `-1`.

- <Status priority='1'>Changed</Status> Changed not to generate 5-minute statistics for the following categories:

  > `db_oracle_active_session`, `db_oracle_xview`, `db_oracle_bind_save`, `db_oracle_lockinfo`, `db_oracle_plan_save`, `db_oracle_pq_tree`, `db_oracle_segments`, `db_oracle_tablespace`, `db_oracle_sga`
