You can group multiple projects for management. A user can own multiple projects. To manage multiple projects, the concept, "Group" has been provided. If there are multiple projects, it is cumbersome to add users or grant permissions for each project. In this case, you can group two projects into a group and manage users by the group.

:::note

- A project can belong to a group.
- A project may not belong to a group.

:::

Project users can inherit group users, and the users can monitor all projects in the group. A user who is granted the role only on the project can monitor only the project, regardless of whether the user is in a group or not.

## Adding a group

1. From the side menu, select <Cmdname sid="BTN07332" className="uitext" /> &gt; ![Add icon](/img/ico-add.svg) <Cmdname sid="BTN06697" className="uitext" />.

   <ImgLang img="group-add.png" desc="Add Group" />

2. On the <Cmdname sid="side_addGroup" className="uitext" /> screen, enter the name of the group to create in <Cmdname sid="group_name" className="uitext" />.

3. Enter the group description in <Cmdname sid="group_description" className="uitext" />. The <Cmdname sid="group_description" className="uitext" /> field is optional.

4. Select a project to add to the group in <Cmdname sid="add_project" className="uitext" />. You can search a desired project for addition.

5. After adding the project, select <Cmdname sid="add_group" className="uitext" />.

:::note

The projects that can be added to the group are the ones with Edit permission or not part of the group.

:::

## Adding a project into a group

1. On the upper right of a project entry in the <Cmdname sid="side_projectList" className="uitext" /> of Home screen, select ![Configuration icon](/img/ico-setting.svg). 

   <ImgLang img='project-add-group.png' desc='Project Setting' />

2. Select <Cmdname sid="add_to_group" className="uitext" />.

3. Select a group to add a project in.

4. Select <Cmdname sid="add_to_group" className="uitext" />.

## Viewing the project list in the selected group

<ImgLang img='project-select-group.png' desc='Selecting Project Group' />

1. By selecting <Cmdname sid="BTN07332" className="uitext" /> in the side menu, open the group and project list panel.

2. In the panel, select the ![Right arrow icon](/img/right-arrow.svg) button on the right of the group to view.

   Only the projects included in the selected group appear in the list. In this status, you can use the <Cmdname sid="side_groupMember" className="uitext" /> or <Cmdname sid="side_groupManagement" className="uitext" /> menu.

3. To see the full list of projects again, select the ![Home icon](/img/ic-home.svg) <Cmdname sid="TLT07384" className="uitext" /> button at the bottom of the side menu. 

:::note

To see the projects that are not part of the group, scroll down the list panel.

:::

## Excluding projects from the group

### Excluding from Home screen

1. On the upper right of a project entry in the <Cmdname sid="side_projectList" className="uitext" /> of Home screen, select ![Configuration icon](/img/ico-setting.svg). 

   <ImgLang img='remove-project-from-group.png' desc='Removing from Group' />

2. Select <Cmdname sid="delete_from_group" className="uitext" />.

3. If the <Cmdname sid="delete_project_from_group" className="uitext" /> window appears, select <Cmdname sid="ok" className="uitext" />.

### Excluding from group management

You can move projects in a group to other projects or exclude them.

1. By selecting <Cmdname sid="BTN07332" className="uitext" /> in the side menu, open the group and project list panel.

2. In the panel, select the ![Right arrow icon](/img/right-arrow.svg) button on the right of the group to delete.

3. In the side menu, select <Cmdname sid="side_groupManagement" className="uitext" />.

4. In the <Cmdname sid="group_management" className="uitext" /> section, select projects to exclude.

   <ImgLang img="group-exclude-project.png" desc="Group management" />

5. If <Cmdname sid="BTN08282" type="project" replace="({count})" className="uitext" /> is displayed, select the button.

6. When the <Cmdname sid="group_management_move" type="project" className="uitext" /> window appears, in <Cmdname sid="select_group" className="uitext" />, select a group to move or select <Cmdname sid="no_group_projects" className="uitext" />.

7. Select <Cmdname sid="confirm" className="uitext" />.

:::note

If you select <Cmdname sid="no_group_projects" className="uitext" /> in <Cmdname sid="select_group" className="uitext" />, the project is excluded from its group.

:::

## Deleting a group

1. By selecting <Cmdname sid="BTN07332" className="uitext" /> in the side menu, open the group and project list panel.

2. In the panel, select the ![Right arrow icon](/img/right-arrow.svg) button on the right of the group to delete.

3. In the side menu, select <Cmdname sid="side_groupManagement" className="uitext" />.

4. In the <Cmdname sid="group_info" className="uitext" /> section, select <Cmdname sid="delete_group" className="uitext" />.

   <ImgLang img="group-remove.png" desc="Delete group" />

5. See the details of the <Cmdname sid="delete_group" className="uitext" /> message window, and then select <Cmdname sid="delete" className="uitext" />.

:::caution

- To delete a group, the user list is also deleted. 
- In case of the projects in the group, only the group information is deleted. 
- The deleted information cannot be restored.

:::

## Inviting group member {#add-member}

1. By selecting <Cmdname sid="BTN07332" className="uitext" /> in the side menu, open the group and project list panel.

2. In the panel, select the ![Right arrow icon](/img/right-arrow.svg) button on the right of the group to invite members to.

3. In the side menu, select <Cmdname sid="side_groupMember" className="uitext" />.

4. Select <Cmdname sid="invite_member" className="uitext" /> on the upper right of the screen.

5. You can also add members by selecting a sub project in <Cmdname sid="group" className="uitext" />.

6. In <Cmdname sid="user_role" className="uitext" />, set the roles of the user to invite and then select <Cmdname sid="apply" className="uitext" />.

7. In the Email input field, enter the email address of the user to invite, and then select <Cmdname sid="add" className="uitext" />. You can continue to add other users.

8. After adding users to invite, select <Cmdname sid="complete" className="uitext" />.

## Checking members for each project

<ImgLang img="group-member-by-project.png" desc="Group member" />

1. By selecting <Cmdname sid="BTN07332" className="uitext" /> in the side menu, open the group and project list panel. 

2. In the panel, select the ![Right arrow icon](/img/right-arrow.svg) button on the right of the group to check members from.

## Modifying a group member role

1. By selecting <Cmdname sid="BTN07332" className="uitext" /> in the side menu, open the group and project list panel.

2. In the panel, select the ![Right arrow icon](/img/right-arrow.svg) button on the right of the group that includes the member whose role to be modified.

3. In the side menu, select <Cmdname sid="side_groupMember" className="uitext" />.

4. In the <Cmdname sid="side_groupMember" className="uitext" /> list, select ![Edit icon](/img/ico-edit.svg) on the right of the member to modify.

5. If the <Cmdname sid="edit_member_role" className="uitext" /> window appears, set the values for <Cmdname sid="member_role" className="uitext" /> and <Cmdname sid="user_role_per_project_of_group" className="uitext" />.

   - <Cmdname sid="member_role" className="uitext" />: You can set the roles for the group and the projects in the group.
   - <Cmdname sid="user_role_per_project_of_group" className="uitext" />: Permissions of the selected members can be set for each project in the group.

6. After all settings are finished, select <Cmdname sid="apply" className="uitext" />.

:::note

Member role editing can be used only by the users with the <Cmdname sid="member_management" className="b600" /> role. For more information about the member roles, see [the following](project-structure#project-auth).

:::

## Deleting group members

1. By selecting <Cmdname sid="BTN07332" className="uitext" /> in the side menu, open the group and project list panel.

2. In the panel, select the ![Right arrow icon](/img/right-arrow.svg) button on the right of the group that include the member to delete.

3. In the side menu, select <Cmdname sid="side_groupMember" className="uitext" />.

4. In the <Cmdname sid="side_groupMember" className="uitext" /> list, select ![Delete icon](/img/ico-trash-red.svg) on the right of the member to delete.

5. If the message window appears, select <Cmdname sid="delete" className="uitext" />.

:::note

- To delete members from subgroups, click any checkbox in the message window and then select <Cmdname sid="delete" className="uitext" />.

- If you have not the <Cmdname sid="member_management" className="b600" /> role, you cannot delete other members. For more information about member roles, see [the following](project-structure#project-auth).

:::

## Downloading the member list {#downlaodmemberlist}

To download the member list as a CSV format file, keep the following steps:

1. By selecting <Cmdname sid="BTN07332" className="uitext" /> in the side menu, open the group and project list panel.

2. In the panel, select the ![Right arrow icon](/img/right-arrow.svg) button on the right of the group that include the member to delete.

3. In the side menu, select <Cmdname sid="side_groupMember" className="uitext" />.

4. Select ![CSV icon](/img/ic-csv-download-ms.svg) on the upper right of the screen.

Save the member list to the local disk in a CSV format file.

:::note

Member role downloading can be used only by the users with the <Cmdname sid="member_management" className="b600" /> role. For more information about the member roles, see [the following](project-structure#project-auth).

:::
