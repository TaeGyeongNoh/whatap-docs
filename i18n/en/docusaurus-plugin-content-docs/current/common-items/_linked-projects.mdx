If you have created a database monitoring (DPM) project and an application monitoring (APM) project, you can relate the two projects for monitoring the data collected from DPM in the APM project.

<ImgLang img="linked-project-management.png" desc="Correlated Project Management" />

The <Cmdname sid="side_correlatedProjectManagement" className="uitext" /> feature allows you to go beyond product-/equipment-centric monitoring and view monitoring data across multiple projects. This is useful for analyzing how different system components such as applications and databases, interact together. In particular, when performance degradation occurs, you can quickly determine whether it is a hardware problem or an individual application problem, which can significantly reduce the troubleshooting time. Through <Cmdname sid="side_correlatedProjectManagement" className="uitext" />, you can comprehensively analyze performance data at the user system level and obtain more visible insights.

:::note

**Notes before you begin**

Supportable databases: PostgreSQL, Oracle, MySQL, SQL Server, CUBRID, Altibase, Redis, MongoDB

:::

## Adding any linked project {#addlinkproject}

1. Go to <Cmdname sid="side_management" className="uitext" /> &gt; <Cmdname sid="side_correlatedProjectManagement" className="uitext" />.

2. In the <Cmdname sid="TTL07015" className="uitext" /> section, select the <Cmdname sid="project_selection" className="uitext" /> input field.

   <ImgLang img="linked-project-management-add-project.png" desc="Correlated Project Management" />

3. If you see a list of projects, select a project to link to. You can also enter a string to search for matching projects.

4. Select one or more projects and then select <Cmdname sid="add" className="uitext" /> that is active.

The selected project is added in <Cmdname sid="TTL07016" className="uitext" />.

## Checking the metrics of the linked project {#checklinkedproject}

Metrics related to the added project can be checked in the <Cmdname sid="side_analysis" className="uitext" /> &gt; <Cmdname sid="side_hitmap" className="uitext" /> menu. You can also see performance metrics for the linked project while viewing transactions.

This feature is useful when analyzing transactions in the APM project. When you found a slow SQL, it is difficult to determine exactly in which DB the problem occurred in the environment using multiple similar DBs.

<ImgLang img="linked-project-db-hitmap.png" desc="Hitmap" />

1. Set the time and <Cmdname sid="application" className="uitext" /> for viewing the transactions, and then select ![Search Icon](/img/ico-btn-search.png).

2. In the <Cmdname sid="hitmap_transaction" className="uitext" /> section, select <Cmdname sid="BTN08519" className="uitext" /> on the upper right.

3. A new window appears where you can check the metrics for the correlated project.

   <ImgLang img="linked-project-db-instance-dahboard.png" desc="Correlated Project" />

Select the correlated project from the left list and then check the metrics in the <Cmdname sid="TTL08442" className="uitext" /> section. 

<ImgLang img="linked-project-db-instance-dahboard-select-instance.png" desc="Correlated Project" />

To navigate to the dashboard menu for a selected project, select an instance and then click <Cmdname sid="BTN08444" className="uitext" /> on the upper right. Go to <Cmdname sid="side_dashboard" className="uitext" /> &gt; <Cmdname sid="side_dbMonitoring" className="uitext" />.

:::note

- The <Cmdname sid="hitmap_transaction" className="uitext" /> button in the <Cmdname sid="BTN08519" className="uitext" /> section appears only when there is an associated project. For more information about adding a correlated project, see [the following](#addlinkproject).

- You can add a project by selecting <Cmdname sid="TTL07151" className="uitext" /> at the top of the left <Cmdname sid="add" className="uitext" /> list.

- The <Cmdname sid="BTN08444" className="uitext" /> button is disabled when <Cmdname sid="all" className="uitext" /> is selected.

  <ImgLang img="linked-project-db-instance-dahboard-not-allowed-btn.png" desc="Instance Dashboard" />

- Depending on the database product type, you can navigate to the <Cmdname sid="side_analysis" className="uitext" /> &gt; <Cmdname sid="side_dbTrend" className="uitext" /> menu.

:::

## Removing any linked project {#removelinkedproject}

1. Go to <Cmdname sid="side_management" className="uitext" /> &gt; <Cmdname sid="side_correlatedProjectManagement" className="uitext" />.

2. To delete a project to unlink from the <Cmdname sid="TTL07016" className="uitext" /> section, select ![Delete icon](/img/ico-trash.svg) on the utmost right of the list.

3. When the confirmation message appears, select <Cmdname sid="delete" className="uitext" />.

The deleted project is excluded from the <Cmdname sid="TTL07016" className="uitext" /> section.
