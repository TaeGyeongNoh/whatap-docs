- ***Node***: You can check [Node details](node-details), [Object manifest](object-manifest), [Node summary analysis](container-map-target-info#node-summary-analysis) and move to the [Node list](resource-node-list) menu.

- ***Deployment***: You can check the [Object manifest](object-manifest) and go to the [Deployment](deployment-list) list menu.

- ***ReplicaSet***: You can check the [Object manifest](object-manifest). 

- ***Pod***: You can check [Pod start analysis](/kubernetes/pod-init-perform-v2), [Object manifest](object-manifest), [Pod summary analysis](container-map-target-info#pod-summary-analysis) and move to the [Pod list](resource-pod-list) menu. 

- ***ContainerId***: You can check the ***Container info*** and [Container summary analysis](container-map-target-info#container-summary-analysis) and go to the [Container list](resource-container-list) menu.

- ***Image***: You can check the ***container image*** and go to the [Container image](container-images) menu. 

- ***ServiceName***: You can check the [Object manifest](object-manifest) and go to the [Service list](service-list) menu. 

- ***Volume***: You can see the volume and mount path used by the container.

  - You can see the name of the ![Leader line](/img/char-a.svg) volume used in the Pod and the type of the ![Leader line](/img/char-b.svg) volume. 

    > e.g. ***Volume*** ![Leader line](/img/char-a.svg)`containerddomainsocket` ![Leader line](/img/char-b.svg)`HostPath` 

  - When the ***HostPath*** volume type is used, you can see the file path of the ![Leader line](/img/char-c.svg) host system. This path connects the file system between the Pod and the host system.

    > e.g. ***Host Path*** ![Leader line](/img/char-c.svg)`/run/containerd/containerd.sock`
