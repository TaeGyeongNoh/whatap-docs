You can check the node information of the current project in a list.

- Real-time data is updated every 10 seconds.

- It provides the functions for node status summary, comparison, filtering, and _CSV_ file download.
  > CSV file name, _kubernetes\_node\_list\_\{pcode}_\{timestamp}.csv_

## Basic screen guide

<ImgLang img='k8s-resource-node-list-m.png' desc='Node list basic screen' />

You can see data by selecting a desired option at the top of <Cmdname sid="node_list" className="uitext" />.

- You can see the number of all nodes and the numbers of nodes for each state.

- If you select a desired column, you can search by sorting <Cmdname sid="node_list" className="uitext" /> based on the selected column.

## Comparing

The **_Compare_** option allows you to see a comparison chart of key metrics for multiple nodes. When you select the **_Compare_** check boxes of the nodes to view in the list, a window comparing details for the nodes appears on the right as shown below.

## View Detailed {#node-list-details}

Node details can be checked through the **_Detail_** option for details on the selected node. If you select the ![Detail icon](/img/ico-detail-view.svg) icon in the **_Detail_** column of the node to view in the list, the detailed information window for the node appears on the right. 

### <Cmdname sid="TTL08441" className="uitext" type="node" />

It provides the summary for configuration of the node selected in the <Cmdname sid="TTL08441" className="uitext" type="node" /> tab and the resource usage. You can comprehensively see the hardware and software components of the node. You can identify the resource usage and the status of each component, and monitor the performance and operational stability.

<ImgLang img='k8s-node-timeline-config.png' />

- **Node system specifications**

  - **_Architecture_**: Hardware architecture used by the node.

  - **_Image_**: Information about the operating system image that the node is running on.

  - **_CapacityMemory_**: Total memory capacity of the node.

  - **_CapacityCpu_**: Total CPU capacity of the node.

- **Software configuration** 

  - **_Container Runtime_**: Version of the container runtime used on the node. 

  - **_kubelet_**: Version of the Kubernetes kubelet running on the node. 

  - **_kube-proxy_**: Version of the kube-proxy that manages the network features. 

- **Resource usage status** 

  - **_CPU_**: CPU usage of the node.

  - **_Memory_**: Memory usage of the node.

  - **_Disk_**: Disk capacity and usage rate on the node.

  - **_Network_**: Network throughput 

:::note 

- If you click CPU, memory, disk, or network, the <Cmdname sid="metrics" className="uitext" /> tab appears that can display the usage trend for the query period. 

- By clicking **_AVG_** and **_MAX_**, you can see the average usage and maximum usage for the query period.

:::

:::note 

The **_Node configuration_** tab is available in case of a cluster project (<code class='my'>CP</code>). 

:::

### <Cmdname sid="resource" className="uitext" />

You can see the ***Pods*** scheduled for the node selected in the <Cmdname sid="resource" className="uitext" /> tab and the lists of ***containers*** and ***applications*** running in those Pods. The information reflects resources for 1 minute based on the end point of the selected time zone. This allows you to more clearly understand the node status at the time point and the operation status of each Pod, container, or application.

<ImgLang img='k8s-node-timeline-resource.png' />

### <Cmdname sid="metrics" className="uitext" />

You can see the resource usage trend of the node selected in the <Cmdname sid="metrics" className="uitext" /> tab. You can check the usage changes in ***CPU***, ***memory***, ***network***, and ***disk*** based on the average (***AVG***) and maximum (***MAX***) values during the query period.

<ImgLang img='k8s-node-timeline-metrics.png' />

- **_AVG_**: Average usage for the query period.

- **_MAX_**: Maximum usage for the query period.

- **_Total_**: Total traffic usage across multiple network interfaces.

- **_In/Out_**: Inbound (receiving) and outbound (sending) traffic usage for a specific network interface.

### <Cmdname sid="trace" className="uitext" />

You can see the trace information of the applications running on the node selected in the <Cmdname sid="trace" className="uitext" /> tab.

<ImgLang img='k8s-resource-node-list-trace.png' />

### <Cmdname sid="event" className="uitext" />

You can check ***K8s node events*** and ***WhaTap node events*** that occurred on the node selected in the In the <Cmdname sid="event" className="uitext" /> tab. ***K8s node events*** include important activity logs within the cluster such as changes in node status, while ***WhaTap node events*** provide information about the events triggered based on the user-defined alert criteria.

<ImgLang img='k8s-node-timeline-event.png' />
