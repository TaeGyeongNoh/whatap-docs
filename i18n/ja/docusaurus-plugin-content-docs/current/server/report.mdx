ホーム画面 > プロジェクト選択 > **_レポート_**

WhaTapモニタリングサービスの初期画面でプロジェクトを選択し、プロジェクトメニューの下の **_レポート_**を選択してください。 レポート画面で ![down-arrow](/img/down-arrow.svg) ボタンをクリックすると、日次、週次、月次に分類された様々な分析レポートを確認できます。

## **_日次サーバーレポート_**

**_日次サーバーレポート_**は、1日あたりのサーバーリソース使用量を確認できるレポートです。 開始日、開始時間、終了時間を設定して表示することができます。

### 概要

<ImgLang img='server_daily_report_01.png' desc='日次サーバーレポート概要'/>

プロジェクト名とホスト数、発生した通知件数を概要を確認できます。 ホスト数と通知件数が前日と比較してどの程度変化したかを確認できます。

-   **ホスト数** ：検索期間中のサーバーの総数です。
-   **通知件数**：参照期間中に発生した通知の総数です。

### サーバー一覧

<ImgLang img='server_daily_report_02.png' desc='日次サーバーレポートサーバーリスト'/>

サーバーごとに日々の主要メトリクスを確認できます。 CPU最大値(CPU Max)とメモリ最大値(MEM Max)の場合、それらの数値を発生した時間で同時に表示します。

アラートの数は、7日前、1日前、当日で比較できます。

:::note

CPUとメモリの平均値と最大値が90%以上の場合、<span class='loger'>赤</span>として表示されます。

:::

### アラート履歴

<ImgLang img='server_daily_report_03.png' desc='アラートリスト'/>

閲覧する時間に発生した通知履歴について、発生時間、遅延、通知が発生したサーバー名、通知の原因を確認することができます。
各通知の詳細情報は**_警告通知_** >**_イベント履歴_**メニューから表示することができます。プロジェクトの通知履歴がない場合は、「該当期間にデータが存在しません」というメッセージが表示されます。

:::note

通知のレベルに応じて <span class='loger'>Critical</span>は <span class='loger'>赤</span>、<span class='wer'>Warning</span>は <span class='wer'>オレンジ</span>、 <span class='mstop'>Info</span>は <span class='mstop'>グレー</span>で表示されます。

:::

## **_週次サーバーレポート_**

**_週次サーバーレポート_**は、一週間のサーバーのCPUとメモリ使用量を確認できるレポートです。開始日を設定することができ、開始日を基準として7日前から閲覧できます。
開始日から7日間のデータを表示します。

### 全体リソースの使用率一覧

#### CPU

<ImgLang img='server_weekly_report_cpu.png' desc='週間全体リソースの使用率一覧CPU'/>

閲覧期間中、各サーバーのCPU使用率を日別に表示します。表示される項目は、CPU平均値と最大値95%、最大値があり、各数値はパーセンテージで表示されます。

:::note 

**最大95%**

上位5%のデータを除く最大値を意味します。 サーバー再起動などの理由による一時的に高い数値を除外するためです。

:::

#### メモリ

<ImgLang img='server_weekly_report_memory.png' desc='週間全体リソースの使用率一覧メモリー'/>

検索期間中の各サーバーのメモリ使用量を日別ごとに表示します。表示される項目には、メモリ使用量の平均値と最大値があり、各数値はパーセンテージで表示されます。

#### ディスク

<ImgLang img='server_weekly_report_disk.png' desc='週間全体リソースの使用率一覧ディスク'/>

検索期間中に各サーバーにマウントされたファイルシステムごとのメトリクスです。
マウントする場所と合計ディスク数、使用率を検索され、各数値はパーセンテージで表示されます。

:::note

-   数値が９０％以上の場合、<span class='loger'>赤</span>で表示されます。 
-   グループ名は、`okindName`と表示されます。`okindName`が設定されていない場合は、空白で表示されることがあります。

:::

## **_月間サーバーレポート_**

**_月次サーバーレポート_**は、1か月間のサーバーリソースの使用量と情報を確認できるレポートです。開始日を設定することができ、参照日を基準として30日前から表示できます。

### 概要

<ImgLang img='server_monthly_report_01.png' desc='月次サーバーレポート概要'/>

1 か月間のサーバーの合計リソース使用量を集計して履歴を表示することができます。

-   **全体ホスト**：検索期間中のサーバー総数です。
-   **全体コア数**：検索期間中に使用された全体のコア数です。
-   **全体ディスク**：検索期間中にマウントされたファイルシステムの合計サイズです。
-   **全体通知**：検索期間中に発生した通知の合計通知件数です。

### サーバー情報

<ImgLang img='server_monthly_report_02.png' desc='サーバー情報'/>

クエリ期間中に稼働していたサーバーの IP アドレス、OS 情報、CPU 情報、およびメモリ使用量を表示します。

### アラート内容

<ImgLang img='server_monthly_report_03.png' desc='アラートリスト'/>

検索期間中のすべてのサーバーの通知の合計数を確認でき、サーバーごとに発生したアラートを週単位で表示できます。

### サーバー使用一覧

<ImgLang img='server_monthly_report_04.png' desc='サーバー使用一覧'/>

サーバーごとのCPU平均、最大値、メモリ平均、最大値を日単位で一目で確認できます。

:::note

数値が９０％以上の場合、<span class='loger'>赤</span>で表示されます。 

:::
