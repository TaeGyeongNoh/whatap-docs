## Global Lock情報を確認する{#checkgloballock}

クラスターで構成されたOracleデータベース環境で、Global Lock情報を確認することは、様々な重要なメリットを提供します。 特に、Oracle Real Application Clusters (RAC) 環境で、これらの情報は性能の最適化と問題解決に不可欠です。

クラスターで構成された環境であれば、<Cmdname sid="instance" className="uitext" />でクラスターを選択できます。

<ImgLang img="db-locktree-cluster-chart.png" desc="ロックツリー" />

- ***Global Lock Wait Sessions***

  選択した時間範囲内で発生したGlobal Lockの待機セッション数を時間帯別に視覚化して表示します。 特定の時間にどれだけ多くのロック待機が発生したのか一目で把握できます。 各色は異なるインスタンスを表し、ロックが発生したインスタンスごとに区分されて表示します。

- ***Global Lock Tree***

  ロックに関する詳細情報をツリー構造で確認できます。 ロックが発生したインスタンス、セッションID(<UI>sid</UI>)、セッション状態(<UI>status</UI>) など重要な情報が含まれます。 ロックのホルダータイプ(<UI>holder type</UI>)とロックモード(<UI>lock mode</UI>)、そしてロックの待機タイプ(<UI>waiter type</UI>)を通じて、ロックの原因と影響を迅速に把握できます。

  ツリー構造により、どのセッションがロックを保持しており、どのセッションが待機中なのかを把握できます。

:::note

クラスターを選択した状態で照会できる最大時間範囲は3時間です。

:::

:::tip

- RAC環境では、複数のインスタンスが同じデータに同時にアクセスできます。 Global Lock情報は、これらの同時アクセス状況でデータの一貫性を確保するための主要なメカニズムの1つです。 これをモニタリングすることで、データ一貫性の維持可否を確認できます。

- Global Lock待ち時間は、システムの性能に大きな影響を与える可能性があります。 クラスターの複数のインスタンス間で発生するGlobal Lock競合をモニタリングすると、性能低下の原因となるボトルネック現象を早期に検知できます。 

- Global Lock情報は、トランザクションがクラスター内でどのように処理されるかについての重要な洞察を提供します。 同時性制御が適切に行われないと、トランザクションの待ち時間が長くなったり、デッドロックが発生する可能性があります。

- RAC環境では、インスタンス間の負荷分散が非常に重要です。 Global Lock情報を通じて、どのインスタンスがより多くの競合を経験しているかを把握することができ、これに基づいて負荷を再分配したり、リソース管理を改善できます。

:::
