データベース指標(メトリクス)は、データベースの作業量とリソース使用量を包括的に収集します。 WhaTapモニタリングサービスは、継続的にメトリクスを収集してデータベースのボトルネックを特定し、性能問題の根本原因を特定するのに役立つデータを提供します。

| メトリクス                                  | 説明                                                         |
| -------------------------------------- | ---------------------------------------------------------- |
| `active sessions`                      | クエリが実行されているセッション数                                          |
| `block disk read`                      | ディスクからblockを読み込んだ回数                                        |
| `block gets (CRX)`                     | CR block examineをリクエストした回数                                 |
| `block updates`                        | Redoログを残してblockを修正した回数                                     |
| `connect_error`                        | 接続エラーが発生した回数                                               |
| `consistent block gets`                | CR blockをリクエストした回数                                         |
| `consistent multi gets`                | Multi CR blockをリクエストした回数                                   |
| `cpu`                                  | CPU使用率(%)                                                  |
| `cpu_cores`                            | CPUコア数                                                     |
| `current block gets`                   | Current blockをリクエストした回数                                    |
| `current block gets - no wait`         | Current blockを待たずに取得した回数                                   |
| `dbwr write` - OS                      | DBWRがOSにwriteをリクエストした回数                                    |
| `execute count`                        | SQL文(statement)を実行した呼び出しの合計数                               |
| `fatal count`                          | 致命的なエラーが発生した回数                                             |
| `iowait cpu`                           | CPUの入出力待機に用いた時間比率                                          |
| `lock wait sessions`                   | 現在rowlockを取得する待機中の数                                        |
| `logical reads`                        | メモリから読み込まれたデータブロック数、 `db block gets`と `consistent gets`の合計 |
| `logons cumulative`                    | インスタンスが起動してからのログオン数の合計                                     |
| `long running sessions`                | 長時間実行中のセッション数                                              |
| `multi block disk read`                | ディスクからmulti blockを読み込んだ回数                                  |
| `parse count (hard)`                   | 構文分析の呼び出し合計数(実際の構文分析)                                      |
| `parse count (total)`                  | 構文分析呼び出しの総数(`hard`, `soft`, `describe`)                    |
| `physical reads`                       | ディスクで読み取られたデータブロック数                                        |
| `pq sessions`                          | 並列処理を行っているセッションの数                                          |
| `sorts (memory)`                       | メモリで実行されたsort作業の数                                          |
| `sys cpu`                              | sessionによって使用された`cpu time`の合計                              |
| `the number of user commits performed` | `sys cpu`使用量(%)                                            |
| `total elapse time`                    | データベースインスタンスが最後に再起動されてから、すべてのリクエストの累積合計経過時間                |
| `total sessions`                       | すべてのセッション数                                                 |
| `user cpu`                             | ユーザーが実行するクエリやプロシージャなどの作業を行うCPUの使用量(%)                      |
| `wait sessions`                        | イベントを待機しているセッション数                                          |
| `warning count`                        | すべてのアラート数                                                  |
