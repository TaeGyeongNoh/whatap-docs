## Service 2.7.0

2024年07月24日

<div class='indentTab'>

### 新機能リリーズ

Service 2.7.0リリーズに含まれる新機能を紹介します。

#### `Database`

- <Status priority='1'>New</Status> **Oracle V2**サービスを開始！

  - <Cmdname sid="side_dashboard" className="uitext" />メニューのUIと機能を大幅に改善しました。

  - <Cmdname sid="side_dashboard" className="uitext" />メニューでデータベースの性能の指標情報を確認できます。

  - データベースログを検索し、推移を確認できる<Cmdname sid="side_log" className="uitext" />メニューを追加し、ログ関連の通知を設定できます。

  :::note

  - **Oracle V2**の新機能についての詳細は、[次の文書](../../oracle/monitoring-intro#new-feature)を参考にしてください。

  - **Oracle V2**を利用するためのエージェントの最小バージョンは**1.9.0**です。 エージェント更新の詳細については、[次の文書](../oracle/agent-manage)を参照してください。

  :::

- <Status priority='1'>New</Status> **Oracle Pro**サービスを開始！

  Direct Memory Access(DMA) 技術を活用したアプローチにより、サーバーのメモリに直接アクセスし、必要なデータを抽出します。 この方法は、データベースサーバーへの負荷を最小限に抑えるか、発生させることはありません。 メモリに直接アクセスすることでデータをほぼリアルタイムで収集でき、システムの性能をより細かくモニタリングし、問題発生時に直ちに対応できます。

  :::note

  **Oracle Pro**についての詳細は、[次の文書](../../oracle-pro/monitoring-intro)を参照してください。

  :::

#### `Kubernetes`

- <Status priority='1'>New</Status> Kubernetesクラスター内で実行中のすべてのサービスの状態と情報を総合的に提供する<Cmdname sid="side_kubeServiceList" className="uitext" />メニューを追加

  :::note

  - この機能は、<Cmdname sid="side_workload" className="uitext" /> &gt; <Cmdname sid="side_kubeServiceList" className="uitext" />メニューパスで確認できます。 サービス構成と現状を詳細に把握することで、クラスター内のネットワーク構成を理解できます。

  - <Cmdname sid="side_kubeServiceList" className="uitext" />メニューの詳細については、[次の文書](../../kubernetes/service-list)を参照してください。

  - この機能は、Kubernetesマスターエージェント1.7.8バージョン以降で対応しています。

  :::

- <Status priority='1'>New</Status>ノードの観点でディスクモニタリングのための<Cmdname sid="side_nodeDiskList" className="uitext" /> メニューを追加

  :::note

  - この機能は、<Cmdname sid="side_cluster" className="uitext" /> &gt; <Cmdname sid="side_nodeDiskList" className="uitext" />メニューパスで確認できます。

  - <Cmdname sid="side_nodeDiskList" className="uitext" />メニューの詳細については、[次の文書](../../kubernetes/node-disk-list)を参照してください。

  - この機能は、Kubernetesマスターエージェント1.7.8バージョン以降で対応しています。

  :::
