2023年12月11日

- <Status>Feature</Status>エージェントオプションの追加

  - **trace_sql_error_stack** <span class='type'>Boolean</span>

    既定値`false`

    SQLエラー内容をスタックとして保存(MySQL、MariaDB、PostgreSQL、SQL Server)します。

  - **trace_sql_error_depth** <span class='type'>Int</span>

    既定値`50`

    `trace_sql_error_stack`オプションを適用する場合に保存されるエラーのライン数を設定します。

  - **profile_error_step_enabled** <span class='type'>Boolean</span>

    既定値 `true`

    プロファイルにエラー関連情報を表示するステップを追加できます。

  - **ignore_http_method** <span class='type'>String</span>

    既定値 `PATCH,OPTIONS,HEAD,TRACE`

    設定されたHTTPメソッド(Method)でリクエストされたトランザクション情報は収集しません。 複数を設定するにはコンマ (,)で区切ります。

    ```ini title="whatap.conf"
    # example
    ignore_http_method=OPTIONS,HEAD
    ```

  - **status_ignore** <span class='type'>String</span>

    既定値`Empty`

    無視しようとするHTTPステータスコードを設定できます。

    ```ini title="whatap.conf"
    # example
    status_ignore=404,500
    ```

  - **status_ignore_set** <span class='type'>String</span>

    既定値`Empty`

    HTTPステータスリターンコードの中で無視したいコードをURL単位で設定できます。

    ```ini title="whatap.conf"
    # example
    status_ignore_set=/a/b/c:400,/a/ab/c:404
    ```

  - **transaction_status_error_enable** <span class='type'>Boolean</span>

    既定値 `true`

    HTTP401、403などの通常の応答ではないHTTP状態コードが返された場合、エラーとして処理します。

- <Status>Fixed</Status> MongoDBモニタリングができない問題を修正

- <Status>Fixed</Status> MongoDB Connection情報を保存するように修正
