2024年03月25日

## 共通

<Status>Changed</Status> <code>cpu</code>指標で<code>xos2</code>指標を廃止し<code>xos</code>指標を百分率値に変更

## MongoDB

<Status>Feature</Status> MongoDB <code class="my">V2</code>追加

- カテゴリ名の統一

  - <code>agentinfo</code> → <code>db_agentinfo</code>
  - <code>session</code> → <code>db_mongo_active_session</code>
  - <code>stat</code> → <code>db_mongodb_counter</code>
  - <code>dbsize</code> → <code>db_dbsize</code>

- 基本指標の簡素化

  <code>v2=true</code>に設定すると<code>serverStatus</code>値のうち<code>opcounters</code>、<code>globalLock</code>、<code>extra_info</code>、<code>connections</code>、<code>network</code>、<code>mem</code>、<code>opLatencies</code>、<code>asserts</code>指標のみを収集・保存

  :::note

  <code>v2=false</code>が既定値であり、v1と同様に<code>serverStatus</code>結果をすべて収集します。

  :::

- 指標追加オプション：<code>category=network,repl,indexStats…</code>などserverStatusのoutputを追加で指定すると追加収集が可能

- データベースサイズの収集情報の追加：<code>index size</code>、<code>storage size</code>、<code>total size</code>収集

- Collection sizeの新規収集：<code>storage size</code>、<code>index size</code>、<code>total size</code>を収集

- これまでアンダーバーで表記していたxos指標(<code>xos_cpu</code>、<code>xos_mem</code>など)を製品の統一性のために、括弧に変更した値も一緒に送信

## Redis

<Status>Feature</Status> Redis <code class="my">V2</code>追加

- カテゴリ名の統一

  - <code>agentinfo</code> → <code>db_agentinfo</code>
  - <code>session</code> → <code>db_redis_active_session</code>
  - <code>stat</code> → <code>db_redis_couter</code>

- <code>db_redis_clusterinfo</code>、<code>db_redis_clusternodes</code>、<code>db_redis_master</code>、<code>db_redis_slave</code>カテゴリを追加

- これまでアンダーバーで表記していたxos指標(<code>xos_cpu</code>、<code>xos_mem</code>など)を製品の統一性のために、括弧に変更した値も一緒に送信
