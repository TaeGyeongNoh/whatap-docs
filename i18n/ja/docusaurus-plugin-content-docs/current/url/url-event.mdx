ホーム画面 &gt; プロジェクト選択 &gt; <Cmdname sid="side_event" className="uitext" /> &gt; <Cmdname sid="side_eventSetting" className="uitext" />

URLモニタリングは、イベント設定を通じてメール形式のアラート通知を提供します。モニタリング中のプロジェクト画面では、リアルタイムで発生するイベントについても通知を提供します。 登録されたURLのイベントを対象に、次のように個別に設定ができます。 

1. イベントリスト画面で編集するURLの![修正アイコン](/img/ico-edit.svg) <Cmdname sid="edit" className="uitext" />アイコンを選択します。

   <ImgLang img='url-event.png' desc='イベントリスト' />

2. そのURLに対する<Cmdname sid="edit_alert_setting" className="uitext" />画面を確認します。 

   <ImgLang img='url-event-mod.png' desc='URLイベント設定' />

   <Cmdname sid="status" className="uitext" />、<Cmdname sid="elapsed" className="uitext" />などのイベントルールは、モニタリングメトリクスと条件式を組み合わせて設定します。

   - **ステータスイベント**

     ステータスイベントは、選択した<Cmdname sid="status" className="uitext" />コードのオプションによってイベントが発生します。

     <ImgLang img='alert_status.png' desc='ステータスイベント' />

   - **経過時間イベント**

     経過時間イベントは、リアルタイムで収集中の<Cmdname sid="elapsed" className="uitext" />値に、ユーザーが入力したしきい値と条件文に該当する場合、イベントが発生します。

     <ImgLang img='alert_elapsed.png' desc='経過時間イベント' />

:::info

**通知メッセージとは？**

そのルールでイベントが発生した場合に表示するタイトルです。個別のイベントルールに対する通知メッセージを設定できます。設定を適用しない場合、基本的に表示される通知メッセージは**ステータスコード**、**経過時間**、**URL: 詳細**です。

アラート通知に関連してモニタリングプラットフォームごとに対応されているイベントの種類を確認するには、[次の文書](../support-env#support-notice)を参照してください。

:::

:::note

修正したURLが一覧に反映されるまで約1分の時間がかかります。

:::
