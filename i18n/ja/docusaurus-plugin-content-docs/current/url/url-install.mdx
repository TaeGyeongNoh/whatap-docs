WhaTapモニタリングサービスを使用するには、[会員登録](https://www.whatap.io/ko/signup/) 後にプロジェクトを作成して、対応するURLを追加する必要があります。会員登録に関する詳細については、[次の文書](../account/account-manage)を参照してください。

次の動画ガイドを参照してください。

{@youtube: uBa6AtXFlHk}

## プロジェクト生成

URLを追加する前に、まずプロジェクトを作成してください。

1.  [WhaTapモニタリングサービス](https://service.whatap.io)に移動し、ログインします。

2.  プロジェクトを作成するには、画面左側のメニューから **_+ <TR sid='project_add_btn'/>_** ボタンを選択してください。

3.  **_製品カタログ画面_**でURLモニタリングプロジェクトを作成してください。

    <ImgLang img='url-project.png' desc='urlモニタリングプロジェクト' />

4.  **_プロジェクト名_**、 **_タイムゾーン_** などの項目を設定してください。

5.  **_データサーバーリージョン_**でWEB-Checker-Regionを選択してください。

6.  すべての設定を完了した後は **_プロジェクト作成_** ボタンを選択してください。

7.  プロジェクト一覧で新しいプロジェクトを確認してください。

:::note

-   **_データサーバーリージョン_**はリージョン(クラウドサービスを提供するためにインストールしたデータセンターのグループ)を意味します。 特定のリージョンを選択すると、そのリージョンに所属するデータセンターにユーザーのデータが保存されます。
-   **_タイムゾーン_**は通知とレポートを生成する基準時間です。
-   複数のプロジェクトをグループで管理する場合は、**_プロジェクトグループ_**からグループを選択するか、グループを追加してください。 グループの詳細については、[次の文書](../project/group)を参照してください。

:::

## URLを追加する

1.  生成されたURLプロジェクトを選択してください。 URLが設定されていない画面が表示されます。

2.  画面右上の **_+ URL追加する_** ボタンを選択してください。 

3.  追加された入力フィールドにURL設定を行ってください。

    <ImgLang img='url-input-field.png' desc='url入力フィールド' />

    -   **_名前_**: URL設定の固有の識別情報として使用されます。重複使用はできません。リセット時に同じ名前で登録する場合は、同じ名前の統計データを同じURLのデータとしてみなします。

    -   **_URL_**: チェック対象のURLを登録します。 呼び出し先URLの完全パスを入力します。http/httpsプロトコルに対応します。

    -   **_確認地域_**: WhaTapが提供するエージェントを選択します。 Seoul、Tokyo、Singapore、Mumbai、Frankfurt、Califoniaリージョンを提供します。 次の確認地域の選択の案内を参照してください。

4.  編集が完了したら、**_保存_**ボタンをクリックして設定情報を保存してください。 

5.  **_モニタリング_** ボタンを選択してモニタリング画面に切り替えてください。

### 確認リージョン選択

今後の接続テストのための追加のリージョン(リージョン)を提供することができます。その他のリージョンについては support@whatap.ioまでお問い合わせください。

## モニタリングする{#url-st}

<ImgLang img='url-monitoring.png' desc='URLモニタリング' />

### 条件指定オプション

ホーム画面からプロジェクトを選択し **_URL一覧_** メニューに移動してください。![number-1](/img/number-01.png)領域の上部にあるオプションを使用すると、表示形式や整列方式を指定したり、検索して一覧を表示することができます。

-   ![一覧ビューアイコン](/img/ic-list.svg)**_一覧ビュー_**：**_URL一覧_**を一覧形式で見ることができます。基本設定は **_一覧ビュー_**です。 

-   ![カードビューアイコン](/img/ic-project.svg)**_カードビュー_**：**_URL一覧_**をカード形式で見ることができます。

-   **_検索_**：入力ウィンドウからURLまたは指定したURLの名前で検索することができます。

![number-2](/img/number-02.png)領域で登録されたURL一覧の最終レスポンスのステータスごとの件数情報を確認し、フィルタリングすることができます。 **_全体_** を見る、 **_9XX_**、 **_サーバーエラー_**、 **_リクエストエラー_**、 **_リダイレクション済み_**、 **_成功_** などに基づいて表示することができます。 基本設定は **_全体_** を見るです。 

### URL一覧のエクスポート、インポート

![number-3](/img/number-03.png) 領域で現在 **_URL一覧_**をExcelファイル形式でエクスポートまたはインポートできます。 

-   ![書き出しアイコン](/img/ico-export.svg) **_書き出し_**：ウェブブラウザーから現在のURL一覧をExcelファイルとしてダウンロードします。

-   ![インポートアイコン](/img/ico-import.svg) **_インポート_**：Excelファイルを選択してURL一覧を読み込みます。 

### URLモニタリングの詳細

![number-4](/img/number-04.png) 領域のURL一覧表で、登録されたURL単位のモニタリングチェック結果を確認することができます。 各カラム項目は、次のとおりです。 URLの登録・編集が必要な場合 ![編集アイコン](/img/ic-edit.svg) **_URLを編集_** アイコンを選択し、編集画面に切り替えてください。

-   **_現在ステータスコード_**

    最終チェックレスポンスコード(Http Status Code)です。各ステータスコードを選択すると、HTTPステータスコードの文書更新に移動します。

-   **_名前/URL_**  

    URL登録時に指定した固有の識別情報とURLです。

-   **_地域_**  

    確認リージョンです。

-   **_経過時間_**  

    20分間のチェック結果(レスポンスコードおよびレスポンス時間) または最終レスポンス時間です。チャート領域からドラッグして時間範囲を表示すると、次のようにタイムゾーンのレスポンスコードおよびレスポンス時間を確認できます。

    <ImgLang img='url-monitoring-dt.png' desc='詳細' />

:::note 

**WhaTap URL Monitoringサービスの使用**

URLモニタリングは、ポリシー変更されるまで無料のサービスとして提供されます。 ユーザーは、別のエージェントをインストールする必要はなく、WhaTapが提供するエージェントおよびサーバーリポジトリを使用します。

:::
