## プロジェクトとは?

WhaTapのモニタリングサービスにおいて、プロジェクトはモニタリング単位を意味します。 プロジェクト単位でモニタリング単位が区分されるとともに、テナント権限も管理します。

第一に、プロジェクトは図のようにモニタリング要素別に区分します。 アプリケーション、サーバー、コンテナなど、モニタリングエージェントごとの基準に従います。

<ImgLang img='project-01.png' desc='プロジェクトの作成'/>

第二に、プロジェクトはモニタリング単位によって区別されます。 Javaアプリケーションでも、Front、Gateway、APIなど、用途別にプロジェクトを区分することができます。 1つのプロジェクト内に用途の区別なくアプリケーションを登録することもできます。

<ImgLang img='demo-project.png' desc='Demo project'/>

:::note

プロジェクトの階層構造の詳細については、[次の文書](../project/project-structure)を参照してください。

:::

## プロジェクト管理 {#management}

ホーム画面 > プロジェクト選択 > **_管理_** > **_プロジェクト管理_**

プロジェクト概要の確認と管理が可能で、次の機能を提供します。 そのプロジェクトの修正権限が必要です。

<ImgLang img='project-manage.png' desc='Project management'/>

- **_プロジェクト名_**：プロジェクト名を修正できます。

- **_プロジェクト説明_**：プロジェクトに関する説明を入力することができます。

- **_プロジェクトコード_**：プロジェクトを識別する数値コードです。

- **_アプリケーションのタイムゾーン_**：タイムゾーンを変更することができます。

- **_APIトークン_**：**_再発行_**ボタンを選択すると、APIトークンを再発行することができます。

- **_アラート通知言語_**：プロジェクトで発生するアラート通知のメッセージの言語を変更できます。

修正した項目がある場合は、**_保存_**ボタンを選択してください。

## プロジェクトの削除

次の2つの方法でプロジェクトを削除できます。

- ホーム画面の全体プロジェクトリストから削除するプロジェクト項目の右上にある![設定アイコン](/img/ico-setting.svg)ボタンを選択してください。 ドロップダウンメニューで**_プロジェクト削除_**を選択してください。

- ホーム画面 > プロジェクト選択 > **_管理_** > **_プロジェクト管理_**メニューに移動してください。 画面の右下にある**_プロジェクト削除_**ボタンを選択してください。

:::caution

ユーザー一覧情報とこれまで収集したデータはすべて削除されます。 削除したデータは再復旧できません。

:::

## プロジェクトメンバーを確認する{#member-list}

ホーム画面 > プロジェクト選択 > **_管理_** > **_プロジェクトメンバー_**

選択したプロジェクトに所属するメンバーの一覧を確認できます。

<ImgLang img='project-member-list.png' desc='プロジェクトメンバー一覧' />

- メンバー名の右側にある<code class='history'>ログイン：...</code>タグは、最新のアクセスです。
- 決済権限を持つメンバー名の右側には、<code class='pay'>決済</code>タグを表示します。
- ユーザーのアカウント名の右側には<code class='my'>マイアカウント</code>タグを表示します。
- WhaTapモニタリングサービスに登録していないアカウントを招待すると、招待対象が登録されるまで名前を**_招待待機中_**と表示します。 招待メールを再送信する場合は、**_再招待_**ボタンを選択してください。

## プロジェクトメンバーを招待する

1. ホーム画面から、メンバーを招待するプロジェクトを選択してください。

2. 画面左側のメニューで**_管理_** > **_プロジェクトメンバー_**を選択してください。

3. 画面右上の**_メンバー招待_**ボタンを選択してください。

4. **_ユーザー権限_**項目で招待するユーザーの権限を設定した後、**_適用_**ボタンを選択してください。

5. **_メール_**項目で招待するユーザーのメールを入力し、**_追加_**ボタンを選択してください。 他のユーザーを追加することができます。

6. 招待するユーザーをすべて追加して、**_完了_**ボタンを選択してください。

:::note

- メンバー権限の詳細については、[次の文書](../project/project-structure#project-auth)を参照してください。
- WhaTapモニタリングサービスに登録していないアカウントには、`新規アカウント`タグを表示し、会員登録メールを送信します。
- 招待を完了する前にキャンセルする招待メンバーがいる場合は、**_招待メンバー一覧_**で![削除アイコン](/img/ico-trash-red.svg)ボタンを選択してください。
- 形式が正しくないメールアドレスを入力する場合は、招待されません。
- すでに招待されてるメンバーは追加で招待することはできません。上位の組織やグループのメンバーは自動的に追加されます。

:::

### WhaTapのサポートを受ける

WhaTap監視サービスの使用中に障害が発生した場合は、WhaTapサポートチームをメンバーとして招待してサポートを受けることができます。 

1. **_プロジェクトメンバー_**メニューから画面右上にある**_WhaTapサポートチームを招待する_**ボタンを選択してください。

2. メンバー招待ウィンドウが表示されたら、**_プロジェクト_**項目でプロジェクトを選択してください。

3. **_完了_**ボタンを選択してください。

:::note

WhaTapサポートチームのアカウントは自動入力されます。 プロジェクトの種類によっては、サポートチームのアカウントは複数ある場合があります。

:::

## プロジェクトメンバーの権限を修正する

1. ホーム画面からプロジェクトを選択してください。

2. 画面左側のメニューで**_管理_** > **_プロジェクトメンバー_**を選択してください。

3. **_プロジェクトメンバー_**一覧で、権限を修正するメンバーの右側にある![編集アイコン](/img/ico-edit.svg)ボタンを選択してください。 

4. **_メンバー権限修正_**ウィンドウが表示されたら、**_メンバー権限_**を設定してください。 そのプロジェクトで許可する権限を設定できます。

5. すべての設定が完了したら、**_適用_**ボタンを選択してください。

:::note

メンバー権限の詳細については、[次の文書](../project/project-structure#project-auth)を参照してください。

:::

## プロジェクトメンバーを削除する

1. ホーム画面からプロジェクトを選択してください。

2. 画面左側のメニューで**_管理_** > **_プロジェクトメンバー_**を選択してください。

3. **_プロジェクトメンバー_**一覧で削除するメンバーの右側の![削除アイコン](/img/ico-trash-red.svg)ボタンを選択してください。

4. メッセージウィンドウが表示されたら、**_削除_**ボタンを選択してください。

:::note

- プロジェクトメンバーの管理権限がない場合、他のメンバーを削除できません。
- プロジェクトメンバーが1人の場合、他のメンバーを削除できません。
- 上位グループから継承されたメンバーは削除できません。
- ユーザーのアカウントは、権限に関係なくプロジェクトから削除できます。
- 決済権限を持つメンバーは、決済権限を譲渡するまでは削除できません。

:::
