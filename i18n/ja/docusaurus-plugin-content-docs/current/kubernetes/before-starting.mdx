## クラスターおよびプロジェクトの構成

WhaTap Kubernetesモニタリングサービスは、シングルユーザーのKubernetesクラスター環境を対象に1つ以上のWhaTapプロジェクトを管理します。

ユーザーのKubernetesクラスターは、WhaTapサービスで1つの**クラスタープロジェクト**として表示します。初めてKubernetes製品のプロジェクトを作成する場合、そのプロジェクトが**クラスタープロジェクト**になります。Kubernetesクラスターに複数のネームスペースが存在し、コンテナ/ポッド(Pod) が多数あり、プロジェクトを分割する場合は、Kubernetes内のネームスペースごとに**ネームスペース(Namespace) プロジェクト**を作成し、分離することができます。

<ImgLang img='k8s-kuber-namespace.png' desc='Kubernetes プロジェクト' />

### ![](/img/number-01.png)クラスタープロジェクト

- Kubernetesクラスター内のすべてのリソース情報をモニタリングするためのプロジェクトです。

- **クラスタープロジェクト**を使用して、マスターエージェントとノードエージェントのインストールを継続します。

- プロジェクトリストで**クラスタープロジェクト**は、プロジェクト名の横に`CP`タグを、一番下に`Kubernetes`タグを表示します。

- **クラスタープロジェクト**の作成の詳細については、[次の文書](install#create-project)を参照してください。

### ![](/img/number-02.png)ネームスペースプロジェクト

- Kubernetesクラスターの下位ネームスペース(Namespace) 単位のプロジェクトです。

- ネームスペース(Namespace) 単位のリソースモニタリングおよびアプリケーションコンテナのモニタリングを実行するためのプロジェクトです。

- プロジェクトリストでは、**ネームスペースプロジェクト**の下に`Namespace`タグを表示します。

- **ネームスペースプロジェクト**管理の詳細については、[次の文書](namespace)を参照してください。
