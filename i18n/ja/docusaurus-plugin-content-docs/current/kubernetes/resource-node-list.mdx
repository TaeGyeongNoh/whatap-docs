:::note

特定のカラムのデータが表示されない場合は、クバネティスノードエージェントが旧バージョンである可能性があります。その場合、十分なデータが収集されるように、クバネティスノードエージェントバージョンを1.3.7以降に更新してください。 

:::

現在のプロジェクトのノードの情報を一覧形式で確認できます。

- 10秒サイクルでリアルタイムのデータを更新します。

- ノード状態のサマリー、比較、フィルター、および_CSV_ファイルのダウンロード機能を提供します。
  > CSV ファイル名、_kubernetes\_node\_list\_\{pcode}_\{timestamp}.csv_

## 比較する

**_比較_**オプションにより、複数のノードの主な指標に対する比較チャートを確認できます。リストから照会するノードの**_比較_**カラムのチェックボックスを選択すると、次のように右側にそのノードの詳細情報を比較する画面が表示されます。

## 詳細を確認 {#node-list-details}

ノードの詳細は**_詳細_**オプションを使用して、選択したノードに関する詳細情報を確認できます。一覧から照会を希望するノード**_詳細_**カラムの![詳細アイコン](/img/ico-detail-view.svg)アイコンを選択すると、右側にそのノードの詳細画面が表示されます。 

import K8snodedt from "../common-items/_k8s-node-details.mdx";

<K8snodedt />

**_ノード詳細_**画面は、上部の4つのチャートとノード内のコンテナの詳細情報で構成されています。ノードセレクタを通じて、照会を希望するノードを選択できます。

<ImgLang img='k8s-resource-node-detail.png' desc='ノード詳細画面' />

- トップチャート

  次の4つのチャート情報を確認できます。

  - **_CPU Usage_**: ノードのCPU使用量を確認できます。

  - **_Memory Usage_**: ノードのメモリ使用量を確認できます。

  - **_Network_**: ノードのネットワークインターフェイスの入出力トラフィックとエラーパケットを確認できます。

  - **_Disk Usage_**: ノードのDisk使用量を確認することができます。 

- コンテナの詳細情報 

  - ノード内のコンテナ詳細情報の上部で、`cpu_quota`、`cpu_request`、`cpu_total_milli`、`mem_limit`、`mem_request`、`mem_working_set`メトリクスの合計を確認できます。 

  - コンテナの詳細情報は、次の内容を提供します。
