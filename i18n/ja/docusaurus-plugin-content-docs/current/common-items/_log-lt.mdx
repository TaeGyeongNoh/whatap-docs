**_ライブテール_**メニューからサーバーコンソールにアクセスすることなく、モニタリング画面でログデータストリームを簡単に確認できます。大量のログの中から必要なログを選別し、ハイライト機能により目的のログをすばやく認識することができます。 

<ImgLang img='log-live-tail.png' desc='ライブテール'/>

**_ライブテール_**メニューから複雑なログでも簡単にアクセスできます。必要に応じて**_フィルター_**や**_ハイライト_**などの機能を使用してリアルタイムで表示することができます。ログデータの検索サイクルは2秒です。{@include: ../common-items/_log-ui-main-term.mdx}

## ![](/img/number-02.png)コンテンツフィルター領域

### コンテンツフィルター

ログのコンテンツの中から、これらのキーワードを含むログのみをフィルタリングします。

<ImgLang img='log-lt-contents-filter.png' desc='コンテンツフィルター'/>

-   入力ウィンドウにフィルターを入力し、![アイコン](/img/ic-search.svg)**_検索_**ボタンをクリックします。クライアントからログデータをフィルタリングし、[number-s3](/img/number-03.png)を参照することができます。 

-   キーワードを含むContentの文字列がハイライトされます。

-   １つまたは複数のキーワードでフィルタリングできます。

### Contentフィルター条件

複数キーワードでフィルタリングをする場合は、次のように記述します。

| 入力文字列                 | 説明                                         | 結果                |
| --------------------- | ------------------------------------------ | ----------------- |
| `a b c`               | 各キーワードをスペースで区切りします。                        | a, b, c           |
| `"Whatap is good."`   | キーワードにスペースを含める場合は、`''`または`""`で囲みます。        | Whatap is good.   |
| `"Whatap\\ is good."` | ""で囲まれたキーワードに、`\`を含める場合は、`\\`と入力する必要があります。 | Whatap\\ is good. |

### ハイライト

目的のキーワードを簡単に識別するためのハイライト機能を提供します。

<ImgLang img='log-lt-highlight.png' desc='ライブテールハイライト'/>

-   ![アイコン](/img/ic-highlight.svg) **_ハイライト_**ボタンをクリックし、キーワードを入力します。

-   入力ウィンドウの左側の**_色_**を選択すると、選択できるカラーメニューが表示されます。 

-   基本的にログレベルによるハイライト(<span class='logwr'>WARN</span>, <span class='loger'>ERROR</span>, <span class='logft'>FATAL</span>)が適用されます。 

-   設定した内容は**プロジェクト単位**で保存されます。
