レポート画面で ![down-arrow](/img/down-arrow.svg) ボタンをクリックすると、日次、週次、月次に分類され、様々なアプリケーション分析レポートを確認できます。

## **_日次アプリケーションレポート_**

**_日次アプリケーションレポート_**は、1日あたりのアプリケーションの性能を数値とチャートで確認できるレポートです。開始日、開始時間、終了時間を設定して、表示することができます。

### 概要

<ImgLang img='app_daily_report_01.png' desc='' />

**概要**では、選択した検索期間のアプリケーション情報を概要が確認できます。**ピークタイムの性能概要**では、その日にスループットが最も高かった時間とその期間の指標を確認することができます。

- **訪問者**(**DAU**, Daily Active User)

  選択した対象期間中に接続したユーザー数です。

- **リクエスト**

  対象期間中に入力されたトランザクションの数です。

- **エラー率**

  エラー件数からトランザクション件数を割った値です。

- **クリティカルなイベント**

  対象期間中に発生したメトリクスイベントのレベルがcriticalであるイベントの数です。

- **アプリケーションサーバー**

  対象期間中にモニタリングしたアプリケーションの数(エージェントの数)です。

- **CPUコア**

  使用期間中のCPUコア数です。

- **APDEX\*100**(Application Performance Index) 

  使用期間中のアプリケーションに対する顧客パフォーマンスの満足度です。

- **スループット**(**TPS**, Transaction Per Second)

  1秒間に処理されたトランザクションの数を意味し、サービスパフォーマンス指標の基準となります。

### 性能推移チャート

<ImgLang img='app_daily_report_dau-process.png' desc='' />

**性能推移チャート**を使用して主要な指標を可視化できます。

**性能推移チャート**に表示されるで指標は、**同時接続ユーザー**、**時間あたりの訪問者数**、**処理量**(TPS)、**平均レスポンス時間**、**CPU使用率**、**ヒープ使用量**、**エラー件数**、**日次トランザクション分布図**があります。各グラフでは、指標が最も高かった時間と数値を確認することができます。

### SQLの平均応答時間TOP10

<ImgLang img='app_daily_report_07.png' desc='' />

利用期間中に使用されたSQLクエリ文の平均レスポンス時間でソートした統計情報です。**_統計/レポート_** > **_統計_** > **_SQL_**ページで各クエリの発生時間と性能を詳細に表示できます。

:::note 

**増加%**  

参照する前日までの3日間のデータを平均した値と参照当日のデータを比較して増加率を計算した数値です。

- 例

  `4月25日00時00分` \~ `23時59分`データ検索 → `4月22日00時00分` \~ `4月24日23時59分`データと比較

  数値が10%以上の場合<span class='ov10'>赤</span>で表示され、5%\~10%の間の場合<span class='ov5ud10'>オレンジ</span>で表示されます。5%未満の増加率は表示されません。

:::

## **_週次アプリケーションレポート_**

**_週間アプリケーションレポート_**は、1週間のアプリケーションの性能を数値とチャートで確認できるレポートです。開始日を設定することができ、利用日の7日前から表示できます。

### 概要

<ImgLang img='app_weekly_report_01.png' desc='' />

- **週間の利用者数**(Weekly Active User)

  1週間にログインしたユーザー数です。

- **週間トランザクション数**

  その週に受信したリクエストの数とレスポンスの応答数です。  

- **APDEX\*100**

  利用期間中のアプリケーションの顧客性能の満足度を示します。    

### 稼働率

<ImgLang img='app_weekly_report_02.png' desc='' />

モニタリング対象別の稼働率を日付別や週間別の平均を確認できます。データがない場合は、**-**と表示されます。データはあっても、その値が小さい場合は、**0**と表示される場合があります。

### 性能推移チャート

<ImgLang img='app_weekly_report_03.png' desc='' />

週間アプリケーションレポートのチャートには、**訪問者数**、**APDEX\*100**、**リアルタイムユーザー**、**トランザクション数**、**レスポンス時間**、**処理量**、**アクティブトランザクション**、 **CPU使用率**、 **ヒープ使用量**が含まれます。

チャートのx軸は日付で、y軸は指標の数値を表します。また、週間データの中で指標値が最も高かった日付とその値をグラフ上のテキストから確認することができます。
グラフの表から、日付別の平均値、最大値、週間平均値などが確認できます。

### トランザクションのTOP10

トランザクションのTOP10では、開始日の1週間のデータと前週のデータを比較することができます。

各トランザクションの詳細情報を表示する場合は、**_統計/報告書_** > **_統計_** > **_トランザクション_**メニューから表示することができます。

:::note 

**例** 開始日4月22日

- 参照 週: `4月22日00時00分`\~`4月28日23時59分`
- 1週間前：`4月15日00時00分`\~`4月21日23時59分`

:::

#### トランザクション数

<ImgLang img='app_weekly_report_04.png' desc='' />

1週間入ってきたトランザクションをトランザクション数に基づいて降順に並べ替えた結果です。

- **トランザクション数**

  アプリケーションにログインしてリクエストが処理され、レスポンスが返された数を意味します。1週間でどのようなサービス(URL)が最もアクティブだったかを確認できる指標です。

- **増減**

  問い合わせの週と問い合わせの前週のトランザクション数を比較した数値で、増減率はパーセンテージで計算される数値です。

#### エラー件数

<ImgLang img='app_weekly_report_05.png' desc='' />

その週に受信されたトランザクションエラー数を降順に並べ替えた結果です。 

- **エラー件数**

  どのサービス(URL)でエラーがもっとも多く発生したか確認できる統計データです。

- **増減**

  問い合わせの週と問い合わせの前週のエラー件数を比較した数値で、増減率はパーセンテージで計算された数値です。

#### 平均応答時間

<ImgLang img='app_weekly_report_06.png' desc='' />

1週間分のトランザクションの平均応答時間を降順に並べ替えた結果です。

  問い合わせの週と問い合わせの前週のレスポンス時間を比較した数値で、増減率はパーセンテージで計算された数値です。

### SQLの平均応答時間TOP10

<ImgLang img='app_weekly_report_08.png' desc='' />

SQLクエリ文の平均レスポンス時間をソートした結果です。各クエリ文の週間件数と平均時間を比較することができます。

- **増減**

  問い合わせの週と問い合わせの前週のエラー発生件数を比較した数値で、増減率はパーセンテージで計算した数値です。

## **_月間アプリケーションレポート_**

**_月次アプリケーションレポート_**は、1か月間のアプリケーションの性能を数値とチャートで確認できるレポートです。開始日を設定することができ、閲覧日の30日前から閲覧できます。

### 概要

<ImgLang img='app_monthly_report_01.png' desc='' />

**概要**は、1か月間の全体**トランザクション件数**、**ユーザー数**、**稼働率**、**アプリケーション数**、**コア数**、**Apdex\*100**の数値を一目で確認できます。

- **月間トランザクション件数**

  1か月間入ってきたトランザクションの数です。

- **月間ユーザー数**(**MAU**、Monthly Active User)

  その月にログインしたユーザー数です。

- **稼働率**

  1か月間の各アプリケーションの稼働率を平均した値です。

- **アプリケーション数**

  1か月間モニタリングしたアプリケーションの合計数です。

- **全体コア**

  1か月間のCPUコアの合計数です。

- **APDEX\*100**(Application Performance Index)

  使用期間中のアプリケーションに対する顧客パフォーマンスの満足度です。

### アプリケーション別の状況

<ImgLang img='app_monthly_report_02.png' desc='' />

モニタリング対象別に**IPアドレス**、**OS情報**、**1か月間の平均稼働率**を確認することができます。

### 性能推移チャート

<ImgLang img='app_monthly_report_03.png' desc='' />

月間の主要な指標をチャートで表示することができます。月間アプリケーションレポートにチャートとした指標は次のとおりです。

- **顧客ロイヤリティ**

  毎週利用するユーザーのうち、毎日何人のユーザーが利用しているかを数値化した値です。

- **毎日の訪問者数**

  このグラフには、その日にログインしたユーザーの数が表示されます。

- **日次トランザクション数**

  日中に発生したトランザクションの数を示したグラフです。

- **1時間当たりのトランザクション数**

  このグラフは、日毎、時間毎に受信したトランザクションの数を示します。

- **時間当たりのエラー件数**

  このグラフは、時間あたりに受信したトランザクション エラーの数を日付別に表示します。

- **アクティブトランザクション**

  このグラフには、時間あたりに受信したアクティブなトランザクションの数が日付別に表示されます。

- **処理量**

  このグラフは、時間ごとに計算され、日ごとに表示される 1 秒あたりのトランザクション スループットを表示します。

- **時間当たりの平均レスポンス時間**

  このグラフは、受信トランザクションの平均応答時間を 1 時間ごとに日付別に表示します。

- **ヒープ使用量**

  このグラフは、日ごとの Java ヒープのメモリ使用量を時間ごとに表示します。

- **CPU SYSの使用率**

  1日当たりのCPU SYS使用率を日付別に表示するグラフです。

- **プロセスCPUの使用率**

  このグラフは、日中の CPU SYS 使用率の割合を日別に示します。

- **APDEX\*100**

  このグラフは、1日あたりの顧客満足度を日付別に表示しるグラフです。

:::note

**APDEX**(Application Performance Index) の計算方法

レスポンス速度によって**満足**、**許容**、**不満**の3段階に分けて、次の計算を行います。  

> APDEX\*100 = (満足したユーザー数 + (許容数 \* 0.5 / 全体ユーザー数)) \* 100

:::
