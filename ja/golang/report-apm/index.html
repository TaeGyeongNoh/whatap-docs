アプリケーションレポート

ホーム画面 > プロジェクト選択 > 統計・レポート > レポート

WhaTapモニタリングサービスの初期画面でプロジェクトを選択し、プロジェクトメニューから 統計/レポート > レポートを選択してください。 レポート画面で ボタンをクリックすると、日次、週次、月次に分類され、様々なアプリケーション分析レポートを確認できます。

日次アプリケーションレポート​

日次アプリケーションレポートは、1日あたりのアプリケーションの性能を数値とチャートで確認できるレポートです。開始日、開始時間、終了時間を設定して、表示することができます。

概要​

概要では、選択した検索期間のアプリケーション情報を概要が確認できます。ピークタイムの性能概要では、その日にスループットが最も高かった時間とその期間の指標を確認することができます。

  • 訪問者(DAU, Daily Active User)

    選択した対象期間中に接続したユーザー数です。

  • リクエスト

    対象期間中に入力されたトランザクションの数です。

  • エラー率

    エラー件数からトランザクション件数を割った値です。

  • クリティカルなイベント

    対象期間中に発生したメトリックイベントのレベルがcriticalであるイベントの数です。

  • アプリケーションサーバー

    対象期間中にモニタリングしたアプリケーションの数(エージェントの数)です。

  • CPUコア

    使用期間中のCPUコア数です。

  • APDEX*100(Application Performance Index)

    使用期間中のアプリケーションに対する顧客パフォーマンスの満足度です。

  • スループット(TPS, Transaction Per Second)

    1秒間に処理されたトランザクションの数を意味し、サービスパフォーマンス指標の基準となります。

性能推移チャート​

性能推移チャートを使用して主要な指標を視覚化できます。

性能推移チャートに表示されるで指標は、同時接続ユーザー、時間あたりの訪問者数、処理量(TPS)、平均レスポンス時間、CPU使用率、ヒープ使用量、エラー件数、日次トランザクション分布図があります。各グラフでは、指標が最も高かった時間と数値を確認することができます。

チャート右上のCSVボタンを使用してデータをExcelファイルとしてダウンロードすることができます。

日次トランザクション分布図では、時間の経過によるトランザクション分布を一目で確認できます。グラフのx軸は各トランザクションの終了時間、y軸はレスポンス時間(秒)を意味します。通常のトランザクションの場合は青で表示され、エラーが発生した場合は、赤で表示されます。

アプリケーション別の状況​

モニタリング対象別にピークタイム性能と日次平均性能を確認することができます。アクティブ件数は、進行中のトランザクション件数を表示します。

トランザクションのTOP10​

トランザクション数、エラー数、平均応答時間が高いトランザクションの上位10件が表示されます。

トップスタックTOP10​

Stack Trace上のStepを基準にStep間の呼び出し頻度を比率として計算した情報を確認することができます。分析 > スタック >トップスタックページでStackの詳細を確認することができます。

SQLの平均応答時間TOP10​

利用期間中に使用されたSQLクエリ文の平均レスポンス時間でソートした統計情報です。統計/レポート > 統計 > SQLページで各クエリの発生時間と性能を詳細に表示できます。

増加%

参照する前日までの3日間のデータを平均した値と参照当日のデータを比較して増加率を計算した数値です。

  • 例

    4月25日00時00分 ~ 23時59分データ検索 → 4月22日00時00分 ~ 4月24日23時59分データと比較

    数値が10%以上の場合赤で表示され、5%~10%の間の場合オレンジで表示されます。5%未満の増加率は表示されません。

週次アプリケーションレポート​

週間アプリケーションレポートは、1週間のアプリケーションの性能を数値とチャートで確認できるレポートです。開始日を設定することができ、利用日の7日前から表示できます。

概要​

  • 週間の利用者数(Weekly Active User)

    1週間にログインしたユーザー数です。

  • 週間トランザクション数

    その週に受信したリクエストの数とレスポンスの応答数です。

  • APDEX*100

    利用期間中のアプリケーションの顧客性能の満足度を示します。

稼働率​

モニタリング対象別の稼働率を日付別や週間別の平均を確認できます。データがない場合は、-と表示されます。データはあっても、その値が小さい場合は、0と表示される場合があります。

性能推移チャート​

週間アプリケーションレポートのチャートには、訪問者数、APDEX*100、リアルタイムユーザー、トランザクション数、レスポンス時間、処理量、アクティブトランザクション、 CPU使用率、 ヒープ使用量があります。

チャートのx軸は日付で、y軸は指標の数値を表します。また、週間データの中で指標値が最も高かった日付とその値をグラフ上のテキストから確認することができます。 グラフの表から、日付別の平均値、最大値、週間平均値などが確認できます。

チャート右上のCSVボタンを使用してデータをExcelファイルとしてダウンロードすることができます。

トランザクションのTOP10​

トランザクションのTOP10では、開始日の1週間のデータと前週のデータを比較することができます。

各トランザクションの詳細情報を表示する場合は、統計/報告書 > 統計 > トランザクションメニューから表示することができます。

例 開始日4月22日

  • 参照 週: 4月22日00時00分~4月28日23時59分
  • 1週間前：4月15日00時00分~4月21日23時59分

トランザクション数​

1週間入ってきたトランザクションをトランザクション数に基づいて降順に並べ替えた結果です。

  • トランザクション数

    アプリケーションにログインしてリクエストが処理され、レスポンスが返された数を意味します。1週間でどのようなサービス(URL)が最もアクティブだったかを確認できる指標です。

  • 増減

    問い合わせの週と問い合わせの前週のトランザクション数を比較した数値で、増減率はパーセンテージで計算される数値です。

エラー件数​

その週に受信されたトランザクションエラー数を降順に並べ替えた結果です。

  • エラー件数

    どのサービス(URL)でエラーがもっとも多く発生したか確認できる統計データです。

  • 増減

    問い合わせの週と問い合わせの前週のエラー件数を比較した数値で、増減率はパーセンテージで計算された数値です。

平均応答時間​

1週間分のトランザクションの平均応答時間を降順に並べ替えた結果です。

  • 平均時間

    アプリケーションサーバーが、リクエスト結果を返すのに掛かった時間です。単位はミリセカンド(ms)です。

  • 増減時間

    問い合わせの週と問い合わせの前週のレスポンス時間を比較した数値で、増減率はパーセンテージで計算された数値です。

トップスタックTOP10​

Stack Trace上のStepをStep間の呼び出し頻度を比率として計算した結果です。そのスタックの1週間前のデータとその週のデータを比較することができます。

  • 増減

    問い合わせの週と問い合わせ前週のスタック呼び出し件数を比較した数値で、増減率はパーセンテージで計算された数値です。

SQLの平均応答時間TOP10​

SQLクエリ文の平均レスポンス時間をソートした結果です。各クエリ文の週間件数と平均時間を比較することができます。

  • 増減時間

    問い合わせ週と問い合わせ前週のSQL処理の平均時間を比較した数値で、増減率はパーセンテージで計算した数値です。

プロジェクトエラー状況​

プロジェクトエラーの状況では、エラーデータを週単位で比較できます。エラーが発生したクラスとエラーメッセージを確認できます。

  • エラー発生件数

    エラークラスとエラーメッセージを合わせた後、降順にソートされた結果です。

  • 増減

    問い合わせの週と問い合わせの前週のエラー発生件数を比較した数値で、増減率はパーセンテージで計算した数値です。

増減率における色の意味

  • 青：増減率がマイナスの場合は、前週に比べて減少したことを意味します。
  • オレンジ：増減率が０％以上１０％未満の場合を意味します。
  • 赤：増減率が１０％以上または、前週になかった新しいデータであることを意味します。

月間アプリケーションレポート​

月次アプリケーションレポートは、1か月間のアプリケーションの性能を数値とチャートで確認できるレポートです。開始日を設定することができ、閲覧日の30日前から閲覧できます。

概要​

概要は、1か月間の全体トランザクション件数、ユーザー数、稼働率、アプリケーション数、コア数、Apdex*100の数値を一目で確認することができます。

  • 月間トランザクション件数

    1か月間入ってきたトランザクションの数です。

  • 月間ユーザー数(MAU、Monthly Active User)

    その月にログインしたユーザー数です。

  • 稼働率

    1か月間の各アプリケーションの稼働率を平均した値です。

  • アプリケーション数

    1か月間モニタリングしたアプリケーションの合計数です。

  • 全体コア

    1か月間のCPUコアの合計数です。

  • APDEX*100(Application Performance Index)

    使用期間中のアプリケーションに対する顧客パフォーマンスの満足度です。

アプリケーション別の状況​

モニタリング対象別にIPアドレス、OS情報、1か月間の平均稼働率を確認することができます。

性能推移チャート​

月間の主要な指標をチャートで表示することができます。月間アプリケーションレポートにチャートとした指標は次のとおりです。

  • 顧客ロイヤリティ

    毎週利用するユーザーのうち、毎日何人のユーザーが利用しているかを数値化した値です。

  • 毎日の訪問者数

    このグラフには、その日にログインしたユーザーの数が表示されます。

  • 日次トランザクション数

    日中に発生したトランザクションの数を示したグラフです。

  • 1時間当たりのトランザクション数

    このグラフは、日毎、時間毎に受信したトランザクションの数を示します。

  • 時間当たりのエラー件数

    このグラフは、時間あたりに受信したトランザクション エラーの数を日付別に表示します。

  • アクティブトランザクション

    このグラフには、時間あたりに受信したアクティブなトランザクションの数が日付別に表示されます。

  • 処理量

    このグラフは、時間ごとに計算され、日ごとに表示される 1 秒あたりのトランザクション スループットを表示します。

  • 時間当たりの平均レスポンス時間

    このグラフは、受信トランザクションの平均応答時間を 1 時間ごとに日付別に表示します。

  • ヒープ使用量

    このグラフは、日ごとの Java ヒープのメモリ使用量を時間ごとに表示します。

  • CPU SYSの使用率

    1日当たりのCPU SYS使用率を日付別に表示するグラフです。

  • プロセスCPUの使用率

    このグラフは、日中の CPU SYS 使用率の割合を日別に示します。

  • APDEX*100

    このグラフは、1日あたりの顧客満足度を日付別に表示しるグラフです。

APDEX(Application Performance Index) の計算方法

レスポンス速度によって満足、許容、不満の3段階に分けて、次の計算を行います。

APDEX*100 = (満足したユーザー数 + (許容数 * 0.5 / 全体ユーザー数)) * 100

チャート右上のCSVボタンを使用してデータをExcelファイルとしてダウンロードすることができます。

トランザクションのTOP10​

トランザクション数​

これは、月に受信するトランザクションをトランザクション数で降順に並べ替えた結果です。

  • トランザクション数

    アプリケーションの受信リクエストが処理され、レスポンスが返された数を意味します。その月にどのようなサービス(URL)が最もアクティブだったか確認できる指標です。

  • 増減

    問い合わせ月と問い合わせ前月のトランザクション数を比較した数値で、増減率はパーセンテージで計算した数値です。

エラー件数​

これは、月に受信したトランザクションをエラー数で降順に並べ替えた結果です。

  • エラー件数

    どのサービス(URL)でエラーがもっとも多く発生したか確認できる統計データです。

  • 増減

    当月と問い合わせ前月のエラー件数を比較した数値で、増減率は、パーセンテージで計算した数値です。

平均応答時間​

これは、受信トランザクションをその月の平均応答時間に降順に並べ替えた結果です。

  • 平均時間

    アプリケーションサーバーが、リクエスト結果を返すのに掛かった時間です。単位はミリセカンド(ms)です。

  • 増減時間

    問い合わせ当月と問い合わせ前月のレスポンス時間を比較した数値で、増減率は、パーセンテージで計算した数値です。

トップスタックTOP10​

Stack TraceのStepをStep間の呼び出し頻度を比率として計算した結果です。スタックの1か月前のデータと当月のデータを比較することができます。

  • 増減

    問い合わせ当月と問い合わせ前月のスタック呼び出し件数を比較した数値で、増減率は、パーセンテージで計算した数値です。

SQLの平均応答時間TOP10​

SQLクエリ文の平均レスポンス時間をソートした結果です。各クエリ文の月間件数と平均時間を比較することができます。

  • 増減時間

    問い合わせ当月と問い合わせ前月のSQL処理の平均時間を比較した数値で、増減率は、パーセンテージで計算した数値です。

増減率における色の意味

  • 青：増減率がマイナスの場合は、前週に比べて減少したことを意味します。
  • オレンジ：増減率が０％以上１０％未満の場合を意味します。
  • 赤：増減率が１０％以上または、前週になかった新しいデータであることを意味します。
