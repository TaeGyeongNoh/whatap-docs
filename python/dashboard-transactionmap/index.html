트랜잭션 맵

홈 화면 > 프로젝트 선택 > 대시보드 > 트랜잭션 맵

트랜잭션 맵 차트는 종료된 개별 트랜잭션의 응답시간을 분포도 형태로 표현한 차트입니다. 히트맵과 동일하게 분포 패턴에 따른 문제점을 발견하고 분석할 수 있습니다. 히트맵은 5분 단위로 트랜잭션을 그룹화해서 보여주지만 트랜잭션 맵은 트랜잭션을 개별로 표시합니다.

히트맵 분석과 다른 점은?

히트맵 차트는 특정 구간 동안의 트랜잭션 발생 건을 합산하여 표시합니다. 발생 수가 많은 곳을 색상으로 차이를 표현하고 있습니다. 히트맵의 경우에는 구간별 합산 데이터를 따로 저장하고 있어 비교적 긴 시간의 추이도 확인할 수 있습니다. 하지만 트랜잭션을 합산해 표시하기 때문에 개별 트랜잭션 정보를 알기 어렵습니다.

  • 장기간, 대용량 트랜잭션 분포도를 분석하려면 히트맵 분석을 이용하세요.(프로젝트 메뉴 > 분석 > 히트맵)

  • 성능 테스트를 하거나 장애 상황에서 1~5분 이내의 트랜잭션 분포도 분석, 수행 시간 제한 없이 트랜잭션 분포도를 확인하려면 트랜잭션 맵을 확인하세요.

  • 히트맵 메뉴에 대한 자세한 내용은 다음 문서를 참조하세요.
  • 히트맵 트랜잭션 차트를 분석하는 방법에 대한 자세한 설명은 다음 문서를 참조하세요.

  • 서버 시간보다 사용자의 PC 시간이 과거로 설정된 경우, 서버 시간과 사용자의 PC 시간이 5분 이상 차이가 날 경우 차트에 데이터가 출력되지 않을 수 있습니다.

트레이스 분석하기​

트랜잭션 맵 위젯의 차트 영역을 드래그하면 세부 트랜잭션 정보를 확인할 수 있는 트레이스 분석 창이 나타납니다. 트랜잭션 목록과 각 트랜잭션 하위의 스텝 정보를 한 번에 확인할 수 있습니다.

트레이스 분석에 대한 자세한 내용은 다음 문서를 참조하세요.

에이전트 선택하기​

프로젝트에 포함된 에이전트를 선택해 에이전트별로 트랜잭션을 확인할 수 있습니다. 중복 선택할 수 있고 전체 선택 체크박스를 선택해 모든 에이전트를 선택할 수 있습니다. 선택한 에이전트에 따라 트랜잭션 맵의 분포도가 변경됩니다.

가장 오른쪽에 버튼을 선택하세요. 에이전트를 서버별, 종류별로 필터링하거나 검색할 수 있습니다.

분류별 에이전트 선택하기​

드롭다운 메뉴에서 에이전트 설정을 통해 분류한 그룹 단위로 에이전트를 선택해 트랜잭션 맵을 확인할 수 있습니다.

  • 에이전트: 개별 에이전트를 선택하거나 모두 선택할 수 있습니다.
  • 에이전트 서버: 에이전트 설정에서 whatap.onode로 분류된 에이전트 목록을 선택할 수 있습니다.
  • 에이전트 종류: 에이전트 설정에서 whatap.okind로 분류된 에이전트 목록을 선택할 수 있습니다.

에이전트 검색으로 선택하기​

검색 입력란에 원하는 에이전트 이름을 입력하세요. 입력 후 바로 적용 버튼을 선택하면 해당 에이전트 목록만 선택됩니다.

과거 데이터 조회하기​

트랜잭션 맵은 실시간 모니터링을 기본 제공합니다. 기간에 설정된 간격으로 트랜잭션을 조회합니다. 과거 데이터를 조회하려면 종료 시간을 선택하세요.

원하는 날짜와 시간을 선택한 다음 확인 버튼을 선택하세요. 선택한 시간을 기준으로 5분 동안의 트랜잭션을 조회할 수 있습니다.

과거 데이터를 조회한 다음 실시간 모니터링으로 전환하려면 실시간 버튼을 선택하세요.

상위 목록 확인하기​

Top 30 섹션에서는 트랜잭션, 도메인, IP 기준으로 트랜잭션 수행 건수가 많은 상위 30개의 목록을 표시합니다.

목록에서 개별 항목을 선택하면 필터 옵션으로 추가되고 트랜잭션 맵 차트에 반영됩니다. 여러 개의 항목을 다중 선택할 수 있습니다.

필터 추가하기​

Top 30 섹션의 목록에서 개별 항목을 선택해 필터를 추가할 수도 있지만 사용자가 원하는 항목을 직접 입력해 추가할 수도 있습니다. 사용자가 입력한 문자열과 일치하는 트랜잭션을 필터링합니다.

  1. 필터 옵션에서 필터 추가 또는 버튼을 선택하세요.

  2. + Add 버튼을 선택하세요.

  3. 트랜잭션, 도메인, IP 중 하나를 선택한 다음 문자열을 입력하세요.

    + Add 버튼을 선택해 필터 항목을 추가할 수 있습니다.

  4. 원하는 필터 항목을 모두 추가했으면 Apply 버튼을 선택하세요.

추가한 필터 항목이 트랜잭션 맵 차트에 반영된 것을 확인하세요.

차트 설정하기​

차트 분석을 위해 사용자가 원하는 조건으로 설정을 변경할 수 있습니다. 트랜잭션 맵 차트 오른쪽 위에 옵션을 이용하세요.

  • 버튼을 선택하면 차트 관련 설정을 수정할 수 있습니다. Y축 지표를 선택할 수 있으며, 최대값을 설정할 수 있습니다. 설정을 적용하려면 Apply 버튼을 선택하세요.

    Y축 지표로는 경과 시간, SQL 시간, DB 연결 시간, HTTP 호출 시간을 선택할 수 있습니다.

  • 또는 버튼을 선택해 Y축을 확대하거나 축소할 수 있습니다.

  • 에러만을 선택하면 차트에 에러 상태의 트랜잭션만 표시합니다.
