WhaTap 모바일 앱

와탭 모바일 앱을 통해 이용 가능한 기능에 대해 안내합니다. 앱을 통해 사용자의 프로젝트 현황을 파악하고, 설정된 이벤트의 알림을 실시간으로 받을 수 있습니다.

모바일 앱을 통해 제공하는 기능은 다음과 같습니다.

  • 대시보드: 프로젝트 목록을 확인하고, 조직별, 카테고리별로 프로젝트 목록을 확인할 수 있으며, 각 프로젝트별 주요 지표들을 모니터링할 수 있습니다.

  • 프로젝트 검색: 프로젝트 이름 또는 코드를 이용해 원하는 프로젝트를 검색할 수 있습니다.

  • 공지사항: 와탭 모니터링 서비스의 주요 공지사항을 확인할 수 있습니다.

  • 통합 알림 기능: 프로젝트별로 설정한 주요 알림을 받을 수 있으며, 지난 알림 목록을 확인할 수도 있습니다.

  • 모바일 앱은 service.whatap.io 서비스를 지원합니다. 설치형 서비스는 지원하지 않습니다.

  • 지원하는 상품 종류는 다음과 같습니다. 향후 업데이트를 통해 지원 범위를 확대하겠습니다.

    • Application: Java, PHP, Python, Node.js, .NET, Go
    • Server
    • Database
    • Kubernetes
    • URL
    • Browser

  • 모바일 앱에서는 프로젝트 그룹 기능을 지원하지 않습니다.

앱 설치하기​

와탭 모바일 앱은 안드로이드와 iOS 환경을 지원합니다. 다음 링크로 이동하거나 QR 코드를 스캔해 앱을 설치할 수 있습니다.

iOSAndroid

iOS 12 버전 이상
Android 5.0 버전 이상

로그인하기​

앱을 처음 사용하는 사용자는 로그인해야 합니다. 와탭 모니터링 서비스에서 가입한 이메일과 비밀번호를 입력한 다음 로그인 하기 버튼을 탭하세요.

사용자 계정이 없다면 다음 문서의 안내에 따라 회원 가입을 진행하세요. 모바일 앱에서는 회원 가입할 수 없습니다.

초기 화면 안내​

로그인을 완료했거나 앱을 처음 실행했다면 사용자가 소속된 프로젝트 목록을 확인할 수 있습니다.

주요 메뉴 알아보기​

모바일 앱은 다음의 주요 메뉴로 구성되어 있습니다. 화면 아래에 메뉴를 탭하면 각 메뉴의 주요 기능을 이용할 수 있습니다.

  • 프로젝트: 사용자가 소속되어 있거나 관리 중인 프로젝트 목록을 확인할 수 있습니다. 개별 프로젝트의 상세 정보를 확인하고, 주요 지표를 조회할 수 있습니다. 또한 검색을 통해 원하는 프로젝트를 필터링할 수 있습니다.
  • 알림: 개별 프로젝트에서 설정된 이벤트의 알림을 최신 순으로 확인할 수 있습니다.
  • 계정: 사용자의 계정 정보를 확인할 수 있으며, 언어 및 알림 수신 변경, OTP 인증을 설정할 수 있습니다.

공지사항 확인하기​

와탭 모니터링 서비스의 공지사항을 확인하려면 화면 오른쪽 위, 모서리에 버튼을 탭하세요.

공지사항 이전 화면으로 돌아가려면 화면 오른쪽 위, 모서리에 버튼을 탭하세요.

상품 종류별 확인하기​

프로젝트 목록을 상품 종류별로 확인할 수 있습니다.

  1. 화면 왼쪽 위, 모서리에 버튼을 탭하세요.

  2. 상품 목록이 나타나면 원하는 항목을 선택하세요.

선택한 상품 항목에 해당하는 프로젝트만 목록에 표시됩니다.

프로젝트 현황 확인하기​

전체 프로젝트를 상품별로 분류한 목록을 확인할 수 있습니다. 초기 화면 위에 버튼을 탭하세요. 각 상품에 해당하는 프로젝트 목록을 확인할 수 있고, 제품 설명 버튼을 탭하면 상품 소개 페이지로 이동합니다.

프로젝트 검색하기​

프로젝트 목록이 많아 프로젝트를 찾기 어렵다면 검색을 통해서 원하는 프로젝트를 찾을 수 있습니다. 검색 입력란에 찾으려는 프로젝트 이름 또는 프로젝트 코드를 입력한 다음 키보드의 완료를 탭하세요.

조직별 확인하기​

조직별로 설정된 프로젝트 목록을 확인하려면 조직에서 비조직을 탭한 다음 원하는 조직을 탭하세요. 선택한 조직에 해당하는 프로젝트 목록만 표시됩니다.

프로젝트​

화면 아래에 프로젝트 메뉴를 탭하세요. 다음의 주요 기능을 제공합니다.

  • 사용자가 소속되어 있거나 관리 중인 프로젝트 목록을 확인할 수 있습니다.

  • 개별 프로젝트의 상세 정보를 확인하고 주요 지표를 조회할 수 있습니다.

  • 조직 단위로 프로젝트 목록을 분류하고 검색을 통해 원하는 프로젝트를 필터링할 수 있습니다.

  • 상품 분류별로 프로젝트를 확인하고 필터링할 수 있습니다.

프로젝트 목록에서 확인할 수 있는 상품 종류는 Application 및 Server, Database, Kubernetes, URL, Browser입니다.

프로젝트 목록 확인하기​

하나의 프로젝트는 다음과 같이 카드 뷰 형태로 표시되며, 프로젝트의 주요 지표, 기본 정보 등으로 구성되어 있습니다.

  • 프로젝트의 가장 위에는 프로젝트의 이름을 표시합니다. 상세 버튼을 탭하면 더 많은 지표를 확인할 수 있습니다.

  • 프로젝트의 가운데 섹션에는 대표되는 지표를 표시합니다. 상품의 종류에 따라 표시되는 지표는 다를 수 있습니다.

    • Application: 액티브 트랜잭션

    • Server: CPU 사용률

    • Database: 액티브 세션

    • Kubernetes: 쿠버네티스 구성 요소에 대한 요약 정보

    • URL: 전체 URL 개수와 정상, 에러 상태의 URL 개수 표시

    • Browser: 페이지 로드 타임라인

  • 프로젝트의 가장 아래에는 상품 종류, 상품 이름, 리전, 프로젝트 코드, 구독 형태 등의 정보를 표시합니다.

대시보드 살펴보기​

프로젝트 목록에서 각 프로젝트의 상세 버튼을 탭하면 대시보드 화면으로 이동할 수 있습니다.

  • 화면 오른쪽 위, 모서리에서 프로젝트 코드 정보를 확인할 수 있습니다.

  • 프로젝트와 관련한 상세 정보 및 에이전트, 서버 정보는 가장 위에 배치된 섹션에서 확인할 수 있습니다.

  • 기준 시간에 표시된 시간 기준으로 10분, 30분, 1시간 동안의 지표를 조회할 수 있습니다. 최근 항목에서 10 Min, 30 Min, 1 Hour 버튼을 탭하세요.

  • 그 외 각 상품별 대표되는 주요 지표들이 배치되어 있습니다. 다음 목록에서 확인할 수 있습니다.

    Appication
    • Active Status: 액티브 트랜잭션의 진행 상태와 각 진행 상태의 액티브 트랜잭션 수를 확인할 수 있는 차트입니다.
    • 액티브 트랜잭션: 에이전트별 진행 중인 액티브 트랜잭션을 확인할 수 있는 차트입니다. 차트에 표시된 색상은 액티브 트랜잭션의 응답 시간입니다. 파랑색(Normal)은 3초 이내, 주황색(Slow)은 3~8초, 빨간색(Very slow)은 8초 이상을 의미합니다.
    • Hitmap: 종료된 트랜잭션의 응답 시간을 분포도로 표현한 차트입니다. TX trace 버튼을 탭하면 트랜잭션 목록을 확인할 수 있는 트랜잭션 리스트 화면으로 이동합니다. 에러가 발생한 트랜잭션 목록을 확인하려면 All 버튼을 탭하세요.
    • Apdex: Application Performance Index(Apdex)는 애플리케이션 성능 지표입니다. 웹 애플리케이션의 고객 만족도를 측정합니다. 사용자 만족도에 대한 지표로 활용할 수 있으며, 0 ~ 1 사이의 값을 갖습니다.
    • TPS: 초당 처리한 트랜잭션 건수입니다.
    • 평균 응답 시간: 애플리케이션 서버가 사용자에게 요청 결과를 반환하는 데 걸리는 평균 시간입니다.
    • 시스템 CPU: 애플리케이션 서버의 CPU 사용량입니다.
    Server
    • CPU Resource Map: 전체 서버들의 CPU 사용량을 나타내는 분포도입니다.
    • CPU Top 5: CPU의 최대 점유율 기준으로 상위 5개의 프로세스 목록입니다.
    • Memory Top 5: 메모리의 최대 점유율 기준으로 상위 5개의 프로세스 목록입니다.
    • Disk Top 5: : 가장 많은 디스크 I/O 사용량 기준으로 상위 5개의 프로세스 목록입니다.
    Database
    • Thread Running Top 10: 인스턴스 별 쿼리를 실행 중인 상위 10개의 세션 수입니다.
    • Thread Connected Top 10: 인스턴스 별 현재 접속되어 있는 상위 10개의 세션 수입니다.
    • Active Sessions: 조회 시간 동안 쿼리를 실행 중인 세션 수를 에이전트별로 표현한 시계열 차트입니다.
    • Thread Connected 차트: 조회 시간 동안 현재 접속된 세션 수를 에이전트별로 표현한 시계열 차트입니다.

    Database 상품의 경우 화면 아래에 목록 메뉴를 탭하면 인스턴스별 지표를 확인할 수 있습니다.

    Kubernetes
    • 쿠버네티스 컨테이너 수: 상태별로 진행 중인 컨테이너 개수를 표시합니다.
    • 쿠버네티스 Pod 수: 상태별로 진행 중인 Pod의 개수를 표시합니다.
    • Pod 수 차트: 조회 시간 동안 상태별로 진행 중인 Pod의 개수를 표현한 시계열 차트입니다.
    • Node CPU: 조회 시간 동안 개별 노드의 CPU 사용률을 표현한 시계열 차트입니다.
    • Node Memory: 조회 시간 동안 개별 노드의 메모리 사용률을 표현한 시계열 차트입니다.

    Kubernetes 상품의 경우 화면 아래에 목록 메뉴를 탭하면 컨테이너 목록을 확인할 수 있습니다. 개별 컨테이너에 대한 상세 정보를 조회할 수 있습니다.

    URL

    사용자가 등록한 URL 목록과 각 URL의 HTTP 상태 코드를 확인할 수 있습니다.

    Browser
    • 페이지 로드 타임라인: 페이지 로드 시간과 로드 단계별 과정을 평균 시간으로 표현한 차트입니다.
    • Pageload Hitmap: 조회 시간 동안 페이지 로드의 응답 시간 분포를 표현한 차트입니다.
    • AJAX Hitmap: 조회 시간 동안 AJAX의 응답 시간 분포를 표현한 차트입니다.
    • 페이지별 에러 건수: 조회 시간 동안 발생한 에러 건수와 페이지별로 발생한 에러 건수 중 상위 N개를 표시합니다.
    • 에러 메시지별 에러 건수: 조회 시간 동안 발생한 에러 메시지 건수와 각 에러 메시지별 오류 수의 합계 중 상위 N개를 표시합니다.

프로젝트별 알림 확인하기​

개별 프로젝트에서 설정한 이벤트로 전송된 알림 목록을 확인할 수 있습니다. 화면 아래에 알림 메뉴를 탭하세요.

프로젝트 설정 확인하기​

개별 프로젝트의 상세 정보와 알림 설정, 사용자 정보 등을 확인할 수 있습니다. 화면 아래에 설정 메뉴를 탭하세요.

  • 프로젝트 정보: 프로젝트의 상세 정보를 확인할 수 있습니다.
  • 알림 설정: 현재 프로젝트의 알림을 설정할 수 있습니다.
    • 수신 설정: 현재 프로젝트에서 설정된 알림 수신 여부를 설정할 수 있습니다.
    • 수신 여부: 전체 알림을 수신할지 위험 수준의 알림만 수신할지 선택할 수 있습니다.
  • 사용자 정보: 현재 사용자의 정보를 확인할 수 있습니다.
  • 프로젝트 사용자 정보: 프로젝트에 소속된 전체 사용자 정보를 확인할 수 있습니다.

통합 알림​

모든 프로젝트의 알림을 확인하려면 모바일 앱 초기 화면에서 알림 메뉴를 탭하세요.

  • 모든 프로젝트의 알림을 최신 순으로 확인할 수 있습니다.

  • 각 알림의 프로젝트 이름을 탭하면 해당 프로젝트로 이동할 수 있습니다.

최대 7일 동안의 알림 내역을 조회할 수 있습니다. 향후 업데이트를 통해 조회 범위를 확대할 예정입니다.

계정 설정​

모바일 앱 초기 화면에서 계정 메뉴를 탭하세요.

계정 메뉴에서 제공하는 기능은 다음과 같습니다.

  • 사용자 계정 정보 확인, 언어 및 조직 설정

  • 알림 수신 설정

  • OTP 인증 기능

앱 기본 언어 변경하기​

모바일 앱의 기본 언어를 변경하려면 언어 항목에서 선호하는 언어를 탭하세요.

기본 조직 설정하기​

프로젝트 메뉴에서 표시되는 프로젝트 목록을 조직에 해당하는 프로젝트 목록으로 기본 설정할 수 있습니다. 조직 항목에서 선호하는 조직을 선택하세요.

알림 수신 설정하기​

모바일 앱을 통해 알림을 수신하거나 수신 설정을 해제할 수 있습니다. 알람 수신 설정 섹션에서 항목을 탭하여 원하는 설정을 적용하세요.

OTP 인증하기​

와탭 모바일 앱을 통해 OTP 인증을 설정하고 웹 서비스에서 로그인 시 입력할 수 있는 OTP 인증키를 제공합니다.

  1. 와탭 모니터링 서비스로 로그인하세요.

  2. 화면 오른쪽 위에 프로필 아이콘을 선택하세요.

  3. 드롭다운 메뉴가 나타나면 계정 관리 메뉴를 선택하세요.

  4. 계정 정보 메뉴로 이동하면 페이지를 아래로 스크롤하여 멀티팩터 인증 섹션으로 이동하세요.
  5. OTP 키 (재)생성 버튼을 선택하면 QR 코드가 표시됩니다.

  6. 와탭 모바일 앱에서 계정 메뉴로 이동하세요.

  7. OTP 섹션에서 QR 버튼을 탭하세요.

  8. 카메라가 실행되면 웹 페이지에 표시된 QR 코드를 스캔하세요.

  9. 와탭 모바일 앱에 OTP 키가 표시됩니다.

  10. OTP 키를 웹 페이지에 입력하고 OTP 키 검증 버튼을 선택하세요.

OTP 인증 과정을 완료합니다. 이후 와탭 모니터링 서비스로 로그인한 다음 OTP 인증키를 요구하면 모바일 앱에서 제공하는 OTP 키를 입력하세요.
